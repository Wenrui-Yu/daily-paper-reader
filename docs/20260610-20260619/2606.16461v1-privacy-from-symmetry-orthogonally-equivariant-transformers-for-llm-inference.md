## 摘要
本地运行大型语言模型通常不切实际，导致敏感文本的推理被迫交给第三方提供商。分割推理通过将令牌保留在客户端、仅发送隐藏表示来部分缓解这一问题，但这些表示仍可通过针对公共嵌入表的最近邻搜索恢复。我们提出了一种正交混淆方法，客户端在传输前将嵌入乘以一个秘密正交矩阵。为了在任意旋转下实现正确推理，我们引入了ConjFormer，这是一种通过轻量级归一化变化（标量RMSNorm）以及所有线性权重的逐块正交共轭而精确具有O(d)等变性的Transformer变体。因此，服务器完全在旋转基中执行整个前向传播，从未观察到未旋转的隐藏状态。在PubMed上微调的GPT-2和Llama 3.2 1B模型上的实验表明，正交混淆消除了直接的余弦最近邻反演，并将令牌恢复从超过35%的top-10降至最多1.3%，而微调后困惑度仅增加0.4%。这些结果表明，在架构层面强化对称性可以为隐私保护的LLM推理提供一种实用防御，无需噪声注入或复杂的密码学机制。

## Abstract
Running large language models locally is often impractical, pushing inference on sensitive text to third-party providers. Split inference partially mitigates this by keeping tokens on the client and sending only hidden representations, but these representations can still be recovered via nearest-neighbor search against the public embedding table. We propose an orthogonal obfuscation procedure in which the client multiplies embeddings by a secret orthogonal matrix before transmission. To enable correct inference under arbitrary rotations, we introduce ConjFormer, a transformer variant that is exactly $\mathrm{O}(d)$-equivariant via a lightweight normalization change (scalar RMSNorm) together with blockwise orthogonal conjugation of all linear weights. As a result, the server performs the full forward pass entirely in the rotated basis and never observes unrotated hidden states. Experiments on GPT-2 and Llama 3.2 1B models fine-tuned on PubMed show that orthogonal obfuscation eliminates direct cosine nearest-neighbor inversion and reduces token recovery from over 35% top-10 to at most 1.3%, while increasing perplexity by only 0.4% after fine-tuning. These results indicate that enforcing symmetry at the architectural level can provide a practical defense for privacy-preserving LLM inference without noise injection or heavy cryptographic machinery.

- **背景**：大型语言模型（LLM）本地部署成本高昂，推理常外包给第三方云服务。客户端仅发送隐藏表示（分割推理），但这些表示仍可通过针对公开嵌入表的最近邻（NN）搜索重建原始令牌，导致隐私泄露。
- **动机**：现有防御方案（如差分隐私噪声、同态加密、可信执行环境）要么大幅降低模型质量，要么计算开销过高，难以实用。因此需要一种轻量级、无噪声、不依赖密码学硬件的保护方法。
- **核心问题**：如何在保持推理准确性的前提下，防止服务器从隐藏表示中恢复客户端输入令牌。
- **整体含义**：本文证明通过强制Transformer架构对正交群$\mathrm{O}(d)$具有等变性（equivariance），客户端可以用秘密正交矩阵混淆嵌入，服务器仅在旋转基中执行计算，从而从根本上消除直接的嵌入反演攻击，为隐私保护LLM推理提供了一种新的实用范式。

### 2. 论文提出的方法论

**核心思想**  
客户端将输入嵌入乘以一个秘密正交矩阵$U$（$U \in \mathrm{O}(d)$）后发送给服务器；服务器使用经过**逐块正交共轭**后的模型权重，在整个前向传播中保持与原始模型等价，但所有隐藏状态均在旋转后的基中表示，客户端最终将输出反旋转后采样。这使得服务器从未接触未旋转的隐藏状态，直接NN反演在原始嵌入坐标中不再可行。

**关键技术细节**

1. **架构修改以实现$\mathrm{O}(d)$等变性**  
   - 将标准LayerNorm或RMSNorm替换为**标量增益的RMSNorm**（单个可学习标量$\gamma$），因为$\|xU^\top\|_2 = \|x\|_2$，故RMSNorm与正交变换可交换。这是唯一必要的架构改变。
   - 位置编码：绝对位置嵌入和ALiBi兼容；RoPE需将每个注意力头的正交矩阵$O_b$限制为与RoPE旋转矩阵可交换的块对角SO(2)矩阵（即$O_b$由$2\times2$旋转矩阵组成），才能保持等变性。

2. **权重共轭（Weight Conjugation）**  
   - 对每个Transformer块$b$，引入三个正交矩阵：$U$（全局，用于输入/输出），$O_b, R_b$（单块，作用于注意力头），以及FFN内部的置换矩阵$P_b$。
   - 共轭规则（以PyTorch线性层$\mathrm{Linear}(X; W, b) = XW^\top + b$为例）：
     - 查询：$\hat{W}_Q^b = O_b \tilde{W}_Q^b U^\top$，$\hat{b}_Q^b = \tilde{b}_Q^b O_b^\top$
     - 键：$\hat{W}_K^b = O_b \tilde{W}_K^b U^\top$
     - 值：$\hat{W}_V^b = R_b \tilde{W}_V^b U^\top$
     - 输出投影：$\hat{W}_O^b = U \tilde{W}_O^b R_b^\top$
     - FFN第一层：$\hat{W}_{\text{FFN},1}^b = P_b^\top \tilde{W}_{\text{FFN},1}^b U^\top$
     - FFN第二层：$\hat{W}_{\text{FFN},2}^b = U \tilde{W}_{\text{FFN},2}^b P_b$
   - 这些共轭确保：若输入为$XU^\top$，则输出为$\text{original\_output} \cdot U^\top$，即功能完全等价。

3. **推理流程**  
   - 离线：客户端基于公开预训练模型，经微调后获得细调权重$\tilde{W}$；采样秘密正交矩阵集合；执行共轭得到$\hat{W}$，将$\hat{W}$和旋转后的LM头（或仅服务器端权重）上传至服务器，同时保留未旋转的嵌入表和LM头在本地。
   - 在线：客户端对每个令牌嵌入$x$计算$xU^\top$后发送；服务器使用$\hat{W}$执行所有Transformer块，得到$\hat{X}_B = X_B U^\top$；客户端接收后乘以$U$恢复$X_B$，并通过本地LM头采样下一个令牌。

**理论保证**  
Lemma 2.1 证明每个共轭块满足$\hat{F}_b(XU^\top) = F_b(X) U^\top$，即块级功能等价。Lemma 2.2 证明标量RMSNorm是$\mathrm{O}(d)$等变的。

### 3. 实验设计

- **数据集**  
  - 预训练：OpenWebText（约20亿tokens）。  
  - 微调：PubMed摘要数据集（医学文本）。  
  - 攻击评估：使用PubMed微调后的校验集计算令牌恢复准确率。

- **Benchmark**  
  - 主指标：**验证困惑度（Validation Perplexity, PPL）** 衡量模型质量。  
  - 攻击指标：**Top-1/Top-10/Top-100令牌恢复准确率**（即真实令牌出现在攻击者预测列表中的比例）。

- **对比方法**  
  - 原始非等变模型（Original）：采用**最近邻（NN）嵌入反演**（余弦相似度匹配公开嵌入表）。  
  - 混淆模型（ConjFormer）：评估三类攻击：  
    - **范数攻击**：利用正交变换保范数，直接匹配令牌范数。  
    - **ALS–Procrustes权重重排**：利用查询/键投影的共轭关系，交替最小二乘估计$U$和$O_b$。  
    - **Gram-based权重重排**：利用块间共享的$U$，通过Gram矩阵$\hat{G}_b = (\hat{W}_K^b)^\top \hat{W}_Q^b = U (\tilde{W}_K^b)^\top \tilde{W}_Q^b U^\top$，用黎曼梯度下降联合估计$U$。该攻击可聚合所有块的信息，更强。  
  - 附加分析：**注意指纹攻击**（利用注意力logits作为令牌标识的统计匹配）。

### 6. 论文的主要结论与发现

1. **正交混淆消除直接NN反演**：在原始模型上，NN攻击Top-10准确率高达35–47%，而在混淆模型上，最强的Gram权重重排攻击仅达到≤1.3%（微调后甚至更低）。  
2. **质量几乎无损**：等变模型预训练PPL与原始模型差距<2%，微调后差距<1%（甚至有时更好）。  
3. **攻击性质转变**：从简单的嵌入匹配变为需要解决权重对齐问题，而后者因微调导致的权重偏离而变得极其困难。  
4. **注意指纹攻击可被微调破坏**：每次用户微调都会改变指纹，迫使攻击者重建匹配表。  
5. **范数攻击可通过微调嵌入表消除**（将令牌范数随机化）。  
6. ConjFormer可作为现有模型的“retrofit”方法：只需替换归一化层并短时间微调即可恢复性能。

### 7. 优点

- **实用性强**：无需噪声注入或密码学，计算负担仅增加一次离线共轭和在线旋转/反旋转（可忽略）。  
- **架构改动极小**：仅需将归一化层替换为标量RMSNorm，易于应用于已有模型。  
- **理论优雅**：利用对称性原理，精确等变性保证了功能等价，无近似或精度损失。  
- **强大防御能力**：将攻击难度从简单最近邻搜索提升到需恢复高维正交矩阵，且随微调而加剧。  
- **兼容多种位置编码**（绝对、ALiBi、受限制的RoPE）。  
- 实验设计合理，覆盖多种模型和攻击，展示了鲁棒性。

### 8. 不足与局限

- **非正式隐私保障**：方法并非密码学或差分隐私机制，不提供信息论安全性；其安全性依赖于密钥保密、微调嵌入表和LM头的私密性，以及服务器无法获取未旋转表示。  
- **正交不变量泄露**：尽管微调可消除范数泄露，但注意力logits、距离、重复模式等仍可见，可能被利用（如统计匹配）。  
- **RoPE限制**：为保持等变性，需将$O_b$限制为块对角SO(2)矩阵，削弱了混淆的随机性；严格来说，此时群作用为SO(2)而非完整O(d)。  
- **密钥重用风险**：如果同一正交矩阵用于多次会话，或被已知明文攻击，可能泄露。  
- **大模型扩展性未验证**：实验限于1B参数；更大模型是否仍能保持同样效果需验证。  
- **硬件侧信道未考虑**：如内存访问模式、时间攻击等。  
- **缺乏与差分隐私等方法的定量对比**：虽然类型不同，但可提供PPL-准确率权衡的对比。
