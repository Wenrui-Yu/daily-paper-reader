## 摘要
大语言模型作为商业API部署时易受模型提取攻击，而现有防御要么行动过晚，要么降低合法用户的使用体验。我们提出知识陷阱（Knowledge Trap），一种通过蜜罐知识图（Honeypot Knowledge Graph）和面包屑引导探索将提取攻击重定向到低迁移性知识的防御方法。知识陷阱不阻止查询或扰动输出，而是将攻击者有限的查询预算消耗在几乎无下游实用性的知识上，同时保留良性用户的性能。在医疗和金融领域的实验表明，知识陷阱在不降低合法用户准确率的情况下，平均将代理模型一致率降低6.2%，优于那些对用户性能有可衡量影响的现有防御。这些结果表明，防御知识空间遍历是缓解大语言模型提取攻击的一个实用方向。

## Abstract
Large language models deployed as commercial APIs are vulnerable to model extraction attacks, while existing defenses either act too late or degrade utility for legitimate users. We propose \textbf{Knowledge Trap}, a defense that redirects extraction attacks toward low-transferability knowledge through a \emph{Honeypot Knowledge Graph} (HKG) and breadcrumb-guided exploration. Instead of blocking queries or perturbing outputs, Knowledge Trap consumes the attacker's limited query budget on knowledge with negligible downstream utility while preserving benign-user performance. Experiments in medical and financial domains show that Knowledge Trap reduces surrogate Agreement by 6.2\% on average without degrading legitimate-user accuracy, outperforming existing defenses that impose measurable user impact. These results suggest that defending knowledge-space traversal is a practical direction for mitigating LLM extraction attacks.

## 2. 论文提出的方法论

### 核心思想
- 利用攻击者对查询预算最敏感这一弱点，将模型知识分为“任务关键知识”（Kc）和“低迁移性知识”（K0）。防御目标是最大化攻击者预算在 K0 上的分配，同时不干扰合法用户。
- 蜜罐知识图（Honeypot Knowledge Graph, HKG）：将 K0 中的概念组织成有向图，每个节点是一个历史过时或边缘化的知识条目，边表示语义关联的后续追问。节点间全为内部边，无法回退到 Kc 区域。
- 面包屑引导（Breadcrumb）：当检测到可疑行为后，在正常回答末尾附加一个短句，悄悄引入一个 HKG 节点作为下一跳探索目标。攻击者的实体扩展管线会自然地将该节点视为高不确定性候选，从而进入 HKG 并不断在图内遍历。

### 关键技术细节
1. **HKG 构建**：
   - 知识来源：三类内容——历史细节（过时临床惯例）、外围分类（领域相关但无用的分类）、狭隘流程记录（技术上正确但无关当前任务的工作流）。
   - 两阶段过滤：第一关关键词过滤（剔除与评测基准重叠的内容）；第二关用探针模型（LLaMA-3.1-8B-Instruct）微调测试，仅保留微调后下游性能提升 ≤ 0.02 的条目。
   - 图结构：每个节点至少 3 个出度，所有边内部循环。
2. **检测与引诱**：
   - 嫌疑分数 s_t = α + γ·1[长度≥ℓ] + ϕ·关键词信号 + min(η, (t+1)/B·ρ)。三个信号均为 API 层可直接观测的统计特征。
   - 当 s_t > τ 时，激活面包屑注入：从预生成的 20 个模板池中随机选取一个，将当前最“高未访问出度”的 HKG 节点名称填入模板，追加到回答尾部。
3. **自强化诱捕**：一旦攻击者查询了面包屑指向的 HKG 节点，该节点的回答又包含指向其邻居的面包屑，不确定性驱动的获取函数会持续选择这些未见节点，形成循环。

### 算法流程（文字说明）
Algorithm 1 描述了整体防御管线：
- 对每个输入查询 x_t，计算嫌疑分数 s_t；
- 若 x_t 直接命中 HKG 节点，则返回 HKG 预置回答；
- 否则，返回正常受害模型回答；若 s_t > τ，则在回答后追加一条包含 HKG 节点的面包屑；
- 面包屑模板随机从池中抽取，防止模板过滤。

- **主要结论**：知识陷阱在所有基准上平均降低代理一致性 6.2%（从 44.0% 降至 37.8%），且合法用户准确率完全不受影响（误触发率 0%），在所有对比方法中表现最好。

## 7. 优点

- **思路新颖**：首次将模型提取攻击视为知识空间遍历问题，并主动重定向攻击者，而非被动检测或牺牲质量。
- **实用性强**：防御不修改模型参数、不扰动输出，合法用户完全无感知，且检测模块仅依赖轻量级会话级统计（O(1) 额外开销），适合生产部署。
- **理论支撑**：提供了用户无害性（Hoeffding 不等式）和攻击预算消耗下界（基于主动学习获取函数）的正式证明，增强了结果可信度。
- **实验全面**：跨领域（医、金、法）、多预算、多自适应攻击、多基线对比，消融实验完整，结论扎实。
- **模块化设计**：HKG 构建、检测、面包屑生成可独立调整，便于后续扩展或结合其他防御。

## 8. 不足与局限

- **攻击者假设局限**：方法针对基于主动学习 + 实体扩展的攻击管线（如 MEAEQ）有效，对随机采样或分布感知查询生成可能失效。论文自身承认了这一局限。
- **HKG 扩展性**：HKG 需离线构建，且一次构建后无法动态扩展。如果攻击者预算极大（远超 2000 次查询），可能耗尽蜜罐节点后返回 Kc 区域。论文建议未来方向为自适应扩展，但未在本文实现。
- **检测器依赖可被绕过**：虽然自适应攻击实验中查询混淆将检测步从 5 推迟到 14，但未测试更强的时序随机化或完全模仿合法用户的查询模式（如使用真实用户查询分布）。
- **合法用户评估不够细**：仅用误触发率衡量，未测试面包屑偶然出现在正常对话中的影响（如信息过载、话题偏离），也未评估代理模型在复杂任务下的能力。
- **成本分析不完整**：虽提到 HKG 构建成本低（~5 美元 API 费用），但未比较与其他防御（如输出扰动）的长期运行成本。定义指出了不对称成本优势（defender cost = O(1) vs attacker waste = Ω(B)），但缺乏具体数值对比。
- **法律领域验证较弱**：仅使用了一个基准 CaseHOLD，且 Agreement 绝对值下降 6.2%，考虑到法律任务的复杂性，可能需要更多测试集才能充分确认泛化性。
- **未考虑防御的公平性风险**：蜜罐知识图包含正确但过时的信息，若攻击者将蜜罐节点数据用于下游应用（如过时法律程序），可能产生有害后果。论文伦理声明提到了知识公开性，但未深入讨论意外误用。
