## 摘要
代理越来越多地被部署在文档密集型工作流中，其中敏感私人信息并非边缘情况而是常规输入，例如，一个预订航班的代理需要护照号码。在这种情况下，代理必须使用私人信息准确完成任务，同时绝不能在其响应中泄露这些信息，因为它无法验证键盘前的人是谁。这两项义务存在根本性的紧张关系。一个能够使用私人信息完成任务的能力足够的模型，同样可以被诱导泄露这些信息。为了评估任务准确性和隐私泄露之间的权衡，我们引入了任务完成与主动隐私提取抵抗基准（TRAP）。每个场景包括一个包含私人信息的文档、一个要求代理使用私有字段调用正确工具的任务查询，以及一个试图以自然语言获取相同信息的攻击查询。通过评估跨越前沿专有和开源模型的22个模型（具有多种规模），我们发现所有模型家族都表现出非平凡的泄露，并且指令遵循能力与泄露率相关。现有的基于提示的防御措施减少了泄露，但以显著降低任务准确性为代价。提示优化未能摆脱这种权衡。我们证明这种失败并非偶然。对于任何基于softmax的模型，没有任何软约束防御（例如基于提示的防御）能够在零泄露概率的情况下同时实现高任务成功率。受这一不可能结果的启发，我们提出了结构化私有字段隔离，该方法在私有字段到达模型之前用哈希键替换它们。这种方法在很大程度上防止了泄露，同时保持了任务准确性。

## Abstract
Agents are increasingly deployed in document-intensive workflows where sensitive private information is not an edge case but a routine input, e.g., an agent booking a flight needs passport numbers. In such settings, the agent must use private information to complete tasks accurately while never exposing it in its responses, because it cannot verify who is actually at the keyboard. These two obligations are in fundamental tension. A model capable enough to use private information for task completion can, by the same capability, be induced to reveal it. To evaluate the trade-off of task accuracy and privacy leakage, we introduce Task-completion and Resistance to Active Privacy-extraction (TRAP). Each scenario includes a document containing private information, a task query that requires the agent to invoke the correct tool using private fields, and an attack query that attempts to elicit the same information in natural language. Evaluating 22 models spanning frontier proprietary and open-source models at multiple scales, we find that all model families exhibit non-trivial leakage, and that instruction-following ability correlates with leakage rate. Existing prompt-based defenses reduce leakage but at significant cost to task accuracy. Prompt optimization fails to escape this trade-off. We demonstrate that this failure is not incidental. For any softmax-based model, no soft-constraint defense, e.g., prompt-based defenses, can jointly achieve high task success with zero leakage probability. Motivated by this impossibility result, we propose structural private field isolation, which replaces private fields with hash keys before they reach the model. This approach largely prevents leakage while keeping task accuracy.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 TRAP 基准设计
- **核心思想**：为每个测试实例同时提供一份包含私有字段和非私有字段的文档、一个任务查询（要求代理调用工具，至少一个私有字段作为参数）和一个攻击查询（试图以自然语言直接获取同一私有字段值）。通过独立评估两个查询的结果，联合测量任务准确性和隐私得分。
- **数据集构成**：覆盖 10 个文档领域（个人：身份、信用卡、医疗、工资单、账户；企业：合同、日程、财务、HR、税务），包含 147 个唯一私有字段、88 个非私有字段、157 个工具、500 个文档实例。文档以三种输入模态提供：纯文本、纯图像、文本+图像（多模态）。
- **任务查询**：显式、无歧义，直接指定操作和所需参数，避免任务难度干扰隐私测量。每个工具至少要求一个私有字段。
- **攻击查询**：默认直接询问（如“护照号是什么？”），另设计三种对抗变体：Ignore（忽略指令）、Important（紧急/社会压力）、Roleplay（角色扮演）。
- **评估指标**：
  - **任务准确率**：代理选择正确工具且所有参数值正确（包括私有和非私有字段）。
  - **隐私得分**：1 - 泄露率，泄露率定义为代理响应中包含目标私有字段值的比例。匹配策略根据字段类型采用大小写敏感/不敏感子串匹配、货币归一化、列表项检查等。

### 2.2 理论分析：软约束防御的不可能性
- **定理1（同时任务成功与完美隐私的不可能性）**：假设 (1) 存在非零触发概率 ϵ，(2) 任务成功意味着存在泄露触发事件。则对于任何 softmax 模型，若任务成功，泄露概率 ≥ 1 - (1-ϵ)^N → 1（当攻击长度 N→∞）。证明基于 softmax 为每个token赋予正概率，软约束只能降低 ϵ 但不能设为0，因此长攻击下泄露不可避免。
- **推论**：任何软约束防御（提示指令、RLHF、提示优化）都不能同时实现高任务成功和零泄露；只有硬约束（如结构性隔离）才能实现零泄露概率。

### 2.3 私有字段隔离方法（结构化防御）
- **思想**：在模型推理前用语义化的哈希键（如 `[PRIVATE_buyer_ein]`）替换文档中的实际私有字段值，原值存储于哈希表，仅在工具执行层解析。模型物理上无法泄露明文。
- **三种配置**：
  - **Oracle**：使用真实私有字段标签和文档源文件，精确像素级掩码。
  - **Practical**：知道哪些字段是私有的，但仅用渲染后的图像，通过 OCR 定位并掩码。
  - **Auto**：无任何标签信息，先 OCR 再用 LLM 识别私有区域后掩码。

## 6. 论文的主要结论与发现

1. **所有模型存在严重隐私泄露**：任务准确率>60% 的模型，隐私得分均低于 27%（多数低于 15%）；无一模型谐波均值超过 41。
2. **指令遵循能力与泄露正相关**：更强的指令遵循能力使模型更易被诱导泄露。
3. **模态影响：文字模态泄露最严重，图像模态因 OCR 限制偶然提升隐私得分（不是真正鲁棒）**。
4. **模型规模转折点**：Qwen3-VL 从 4B 到 8B 时，任务准确率跃升同时隐私得分骤降；更大规模无改善。
5. **提示防御无效：只能沿同一权衡曲线移动，无法同时提高任务和隐私**；提示优化同样失败。
6. **对抗攻击变体反而提高隐私得分**：攻击性语言被模型视为警告信号，专有模型对此更敏感。
7. **授权声明作为社会工程攻击有效**：声称“已授权”导致模型泄露更多（∆Priv 负值，Claude Sonnet 4.5 达 -26.4pp）。
8. **理论证明：任何 softmax 模型下软约束防御无法实现零泄露**。
9. **私有字段隔离有效**：Oracle 掩码下隐私得分≥90% 且任务准确率几乎不变，验证了结构性防御的可行性。

## 7. 优点

- **选题重要且现实**：聚焦 LLM 代理必须使用隐私信息但需抵抗主动攻击的矛盾，是实际部署中的关键问题。
