## 摘要
稠密嵌入为语义搜索和检索增强生成提供动力，但泄露的向量数据库也会泄露其背后的文本，因为嵌入可以被高保真地反演。全同态搜索在原理上可靠，但在百万文档规模下速度过慢，而隐私噪声在保护排名之前就降低了排名质量。我们研究了一条基于不对称性的中间路径：静态文档集合通过几何方式保护——每个向量被SVD截断到低维子空间，并由数据所有者独享的秘密正交变换旋转——而动态查询则通过CKKS进行密码学保护，使得诚实但好奇的服务器永远无法看到查询值或相似度分数。我们证明了任何限定在受保护子空间内的解码器重构误差的紧下界。在一个包含五个编码器的百万文档语料库上，这种保护保持了检索质量（对最强编码器甚至略有提升），产生了线性去噪效果，延迟低于一秒，而现成的反演攻击则崩塌至噪声基底。我们还量化了边界：已知明文攻击者通过正交普鲁克问题，从大约与保留维度数量相当的泄露对中恢复出秘密旋转。同样的不对称几何还可作为大语言模型防火墙的隐私保护语义数据丢失预防原语：仅持有受保护向量的服务器能够以与明文检测器近乎等同的精度检测候选文本是否匹配机密参考语料库，且在文本混淆下性能优雅下降。我们明确说明了限制：查询机密性基于密码学，但文档保护依赖于SVD截断和秘密旋转，这构成了一个经验性混淆层而非密码学原语，这一切都在清晰界定的威胁模型下。

## Abstract
Dense embeddings power semantic search and retrieval-augmented generation, yet a leaked vector database also leaks the text behind it, because embeddings can be inverted with high fidelity. Fully homomorphic search is sound but far too slow at million-document scale, while privacy noise degrades ranking before it protects. We study a middle path built on an asymmetry: the static document collection is protected geometrically - each vector is SVD-truncated onto a lower-dimensional subspace and rotated by a secret orthogonal transform held only by the data owner - while the dynamic query is protected cryptographically under CKKS, so an honest-but-curious server never sees query values or similarity scores. We prove a tight lower bound on the reconstruction error of any decoder confined to the protected subspace. On a one-million-document corpus with five encoders the protection preserves - and on the strongest encoders slightly improves - retrieval quality, a linear-denoiser effect, at sub-second latency, while an off-the-shelf inversion attack collapses to the noise floor. We also quantify the boundary: a known-plaintext attacker recovers the secret rotation by orthogonal Procrustes from about as many leaked pairs as the retained dimension. The same asymmetric geometry doubles as a privacy-preserving semantic data-loss-prevention primitive for LLM firewalls: a server holding only the protected vectors detects whether a candidate matches a confidential reference corpus at near parity with a plaintext detector, degrading gracefully under text obfuscation. We state the limits plainly: query confidentiality is cryptographic, but document protection rests on SVD truncation and a secret rotation that form an empirical obfuscation layer, not a cryptographic primitive, under a clearly delimited threat model.

## 混合隐私感知语义搜索：受限威胁模型下基于SVD截断的文档几何与CKKS加密查询重排序

### 1. 论文的核心问题与整体含义

- **研究动机**：语义搜索依赖于稠密向量（嵌入）。然而，一个核心安全问题是“嵌入泄露等于文本泄露”，因为现有反演攻击能以高保真度从嵌入中恢复原始文本。现有的两种极端防御方案均不理想：全同态加密（FHE）在百万文档规模下速度过慢，差分隐私（DP）在达到有用隐私预算前便已严重破坏检索质量。
- **整体含义**：论文旨在探索一条“中间路径”，提出一种非对称的混合隐私保护方案。该方案的核心思想是区分对待静态的文档集合和动态的查询向量，在**效率**、**准确性**和**隐私**之间寻找一个实用的平衡点。论文并非提出一套可部署的系统，而是对其提出的非对称构造进行科学的理论和实证分析，并明确其适用边界和威胁模型。

### 2. 论文提出的方法论

- **核心思想**：采用不对称的保护策略。
    - **静态文档侧**：通过几何变换进行保护。每个文档向量被**SVD截断 (SVD truncation)** 降维至原始维度的一半（d/2），随后被一个只有数据所有者知道的**秘密正交旋转矩阵R**进行旋转。这是**经验性混淆层**，而非密码学原语。
    - **动态查询侧**：通过密码学进行保护。查询向量在客户端进行**CKKS同态加密**，发送给服务器执行重排序。服务器的计算是**密文-明文 (ct-pt)** 模式，避免了耗时的密文-密文（ct-ct）重线性化步骤，极大地提升了效率。
- **关键技术细节**：
    - **离线阶段**：
        1. 中心化：在文档嵌入（X）上计算全局质心µ，并减去。
        2. SVD截断：计算随机SVD `X ≈ Uₖ Σₖ Vₖᵀ`，取前k个右奇异向量`Vₖ ∈ R^(d×k)`构成投影矩阵，实现降维（k = d/2）。
        3. 秘密旋转：生成一个Haar均匀分布的随机正交矩阵`R ∈ O(k)`，对投影后的向量进行旋转：`v‘ = R Vₖᵀ (v - µ)`。
        4. 公开PQ制品：在旋转后的空间训练乘积量化索引（PQ），生成码本和文档码，供客户端进行本地粗粒度的候选过滤。
    - **在线阶段（七步协议）**：
        1. 编码器：客户端计算查询文本的嵌入`v_q`。
        2. 变换：应用与文档相同的投影和旋转：`q‘ = R Vₖᵀ (v_q - µ)`。
        3. 本地PQ搜索：客户端使用公开的PQ制品进行非对称距离搜索，返回一个短列表的候选文档ID集合I_cand（实验中为40个）。
        4. CKKS加密：客户端将旋转后的查询`q’`加密为`ct_q`。
        5. 服务器重排序：服务器计算每个候选ID对应的密文-明文内积：`ct^(j)_score = ct_q ⊙ v‘_j`。
        6. 解密：客户端将分数密文解密得到`s_j`。
        7. 排序：客户端对解密的分数进行排序，返回Top-K结果。
- **关键公式/引理**：
    - **引理1 (解码器下界)**：对于任何其输出被限制在`span(Vₖ)`内的解码器f，其与原始向量的L2重构误差至少为`||x⊥||₂`（即被SVD丢弃的分量的L2范数）。这量化了SVD截断所带来的信息损失下限。
- **CKKS参数选择**：通过一个可复现的离线基准测试，在安全、准确性和延迟之间选择最优参数：`Npoly=8192`，`[60, 40, 60]`比特的系数模数链，`∆ = 2⁴⁰`。该配置比TenSEAL默认设置快1.7倍。

### 8. 不足与局限

- **文档保护的局限性**：文档保护**不是密码学级的**。一旦威胁模型超出论文定义的`非自适应/未知旋转`范围（如遇到已知明文攻击或自适应学习的解码器），文档保护就会被轻易攻破。
- **实验环境局限**：所有实验均在**单机本地**完成，缺少真实网络延迟和分布式部署下的性能评估。带宽压力（响应体高达11.4 MB）在真实场景中是一个挑战。
- **信息泄露公开**：公开PQ制品会泄露大量邻域结构信息，这是一个已量化的暴露通道。组合访问模式（即哪些候选文档被重排序）也是暴露的。作者建议与PIR/ORAM结合，但未做。
- **通用性验证不足**：集成实验使用**自检索**（self-retrieval），这是一个受控探针，不能完全代表生产级IR基准（如BEIR）的性能。作者在8.6节承认，SVD截断在BEIR上显著降低了`nDCG@10`。
