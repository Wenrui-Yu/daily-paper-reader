## 摘要
物联网和医疗物联网设备的快速普及在医疗和工业环境中引入了关键的网络安全漏洞，这些环境中的资源受限设备需在严格的延迟和数据隐私法规下运行。本文提出了联邦时序图卷积网络结合优势演员-评论家（Federated TGCN-A2C），一种隐私保护的防御架构，集成了四种机制：基于PyG的时序GCN，使用带有全局平均池化的GCNConv层和一个学习到的异常门用于流级威胁分类；基于LSTM的数字孪生生成每个设备的异常分数，通过学习到的sigmoid耦合门控分类器；联邦A2C智能体根据七维状态（包括置信度、熵、异常幅度和流量组成）选择允许、隔离和蜜罐重定向动作；以及增强的蜜罐层，将可疑流量转换为威胁情报，具有自适应阈值。联邦聚合采用EMA平滑的每客户端验证损失作为逆加权FedAvg系数，以在非独立同分布分布下稳定全局模型更新，并每轮使用余弦退火学习率。在CICDDoS 2019和TON-IoT基准测试上评估，该框架实现了99.48%和99.61%的测试准确率，加权F1分数为0.9948和0.9961，在25和10个联邦轮次内收敛，性能比Fed-Inforce-Fusion高出0.21个百分点，同时覆盖了三个额外的攻击类别。所有16个CICDDoS 2019类别的F1至少为0.9237，所有10个TON-IoT类别的F1至少为0.9488，包括严重不平衡的MITM类别。通过SHAP、LIME、Grad-CAM和反事实分析的事后可解释性证实决策基于语义上有意义的流特征，支持临床部署中的监管问责。

## Abstract
The rapid proliferation of IoT and IoMT devices introduces critical cybersecurity vulnerabilities in healthcare and industrial environments where resource-constrained devices operate under strict latency and data-privacy regulations. This paper presents the Federated Temporal Graph Convolutional Network with Advantage Actor-Critic (Federated TGCN-A2C), a privacy-preserving defense architecture integrating four mechanisms: a PyG-based Temporal GCN using GCNConv layers with global mean pooling and a learned anomaly gate for flow-level threat classification; LSTM-based Digital Twins generating per-device anomaly scores gating the classifier via learned sigmoid coupling; a Federated A2C agent selecting among ALLOW, ISOLATE, and HONEYPOT-REDIRECT actions based on a seven-dimensional state capturing confidence, entropy, anomaly magnitude, and traffic composition; and an enhanced honeypot layer converting suspicious traffic into threat intelligence with adaptive thresholds. Federated aggregation employs EMA-smoothed per-client validation losses as inverse-weighted FedAvg coefficients to stabilize global model updates under non-IID distributions, with cosine-annealed learning rates per round. Evaluated on CICDDoS 2019 and TON-IoT benchmarks, the framework achieves 99.48% and 99.61% test accuracy with weighted-F1 scores of 0.9948 and 0.9961, converging within 25 and 10 federated rounds, outperforming Fed-Inforce-Fusion by 0.21 percentage points while covering three additional attack categories. All sixteen CICDDoS 2019 classes achieve F1 of at least 0.9237 and all ten TON-IoT classes achieve F1 of at least 0.9488, including the severely imbalanced MITM category. Post-hoc explainability via SHAP, LIME, Grad-CAM, and counterfactual analysis confirms decisions are grounded in semantically meaningful flow features, supporting regulatory accountability in clinical deployments.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- **四组件融合**：轻量级LSTM数字孪生（提供设备行为异常信号） → 时序GCN（基于PyG，利用GCNConv层+全局平均池化，输出概率） → 联邦A2C智能体（根据七维状态选择ALLOW/ISOLATE/HONEYPOT_REDIRECT动作） → 增强蜜罐（持续收集威胁情报）。
- **联邦聚合**：对时序GCN使用EMA平滑的客户端验证损失逆加权FedAvg；对A2C使用均匀加权（因非IID奖励不可比）。
- **异常门机制**：数字孪生异常分数通过一个学习到的每个类别的sigmoid门控放大分类器logits，增强对新型攻击的检测。

### 关键技术细节
- **数字孪生**：单层LSTM（64隐藏单元），仅用正常流量训练，重建误差作为异常分数。
- **时序GCN**：窗口内每个时间步作为节点，k-NN构建边（k=5），两层GCNConv，隐藏维度128，输出经全局平均池化后由MLP分类；异常门为两层MLP+sigmoid，输出每个类别的缩放因子 \( \hat{\mathbf{y}} = f_{\text{cls}}(\mathbf{h}_{\text{graph}}) \cdot (1 + g(a_i)) \)。
- **A2C状态**：七维向量 \( s = [p_{\max}, \min(H,5.0), \min(a_i,5.0), \mathbb{1}[\ell=c_{\text{normal}}], n_k/50000, r_{\text{normal}}, \sigma_{\text{probs}}] \)。
- **奖励函数**：正确蜜罐重定向（攻击）基础奖励4.0+置信度奖金+异常奖金；正确允许（正常）奖励2.5+置信度奖金；正确隔离（攻击）奖励2.0+置信度奖金；错误动作惩罚。
- **自适应蜜罐阈值**：初始0.70，每10次成功捕获减0.95倍（下限0.40），每3次失败捕获增1.05倍（上限0.90）。
- **联邦聚合权重**：\( w_k(r) = (1/(\ell^{\text{EMA}}_k(r) + \epsilon)) / \sum_j (1/(\ell^{\text{EMA}}_j(r) + \epsilon)) \)，EMA平滑因子α=0.4。
- **余弦退火学习率**：\( \text{lr}_r = \text{lr}_0 \times 0.5 \times (1 + \cos(\pi r / R)) \)。

### 算法流程（三个阶段）
1. **初始化阶段**：构建全局k-NN边索引；为每个客户端训练数字孪生（仅正常流量）；初始化全局GCN和A2C网络；部署蜜罐；设置EMA损失追踪器。
2. **联邦训练阶段**（循环R轮）：计算余弦退火学习率 → 每个客户端并行：下载全局模型 → 本地训练GCN 70 epoch（带类平衡采样和异常门） → 计算验证损失并更新EMA → 切换GCN为评估模式 → 收集200个平衡A2C经验（构建七维状态、采样动作、计算奖励） → 若动作是HONEYPOT_REDIRECT且确认攻击，调整蜜罐阈值 → A2C梯度更新 → 上传模型和EMA损失 → 服务器计算逆EMA损失权重并归一化 → 时序GCN按加权平均聚合，A2C按均匀平均聚合 → 评估全局模型 → 保存最佳checkpoint → 早停（若5轮滚动F1变化小于3e-4且均值大于0.97）。
3. **实时推理阶段**：对每个设备计算数字孪生异常分数 → 时序GCN输出概率 → 构建七维状态 → A2C选择动作 → 执行动作并记录威胁情报 → 根据攻击等级调整资源分配（四级降级策略：Level 0~3对应70/30, 50/50, 30/70, 10/90的医疗/安全资源比例）。
