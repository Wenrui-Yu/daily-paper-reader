## 摘要
在本文中，我们通过引入一种新颖框架来应对联邦学习（FL）中隐私保护训练的挑战，该框架仅对最敏感的隐私特征进行选择性加密，而其余数据及相应模型部分保持未加密。我们提出HADES，一种混合系统，它识别并加密最关键的特征，同时确保隐私保护与计算效率。与遭受高计算开销的全加密FL训练管道不同，HADES通过融合机制整合了加密与未加密的训练管道，实现了加密与明文模型表示之间的无缝交互。为此，我们使用PCA来识别并加密最隐私敏感的特征，这显著降低了FL中重构攻击的成功率。基于这一见解，我们设计了一种混合FL系统，通过多方同态加密（MHE）对选定特征训练端到端加密网络，同时对剩余特征训练明文网络。这两个网络随后通过融合机制进行集成。我们还引入了一种通用打包方案，通过考虑整个神经网络架构来消除冗余旋转。最后，我们证明HADES在通过选择性加密实现隐私保护并优化运行时间的同时，达到了与原始FL相当的准确率。

## Abstract
In this paper, we address the challenge of privacy-preserving training in federated learning (FL) by introducing a novel framework that selectively encrypts only the most privacy-sensitive features while leaving the remaining data and the corresponding model portion unencrypted. We propose HADES, a hybrid system that identifies and encrypts the most critical features, ensuring both privacy protection and computational efficiency. Unlike fully encrypted FL training pipelines, which suffer from high computational overhead, HADES integrates an encrypted and non-encrypted training pipeline via a fusion mechanism, enabling seamless interaction between encrypted and plaintext model representations. To achieve this, we use PCA to identify and encrypt the most privacy-sensitive features, which significantly reduces reconstruction attack success in FL. Building on this insight, we design a hybrid FL system that trains an end-to-end encrypted network via multiparty homomorphic encryption (MHE) on the selected features while simultaneously training a plaintext network on the remaining features. These two networks are then integrated using a fusion mechanism. We also introduce a general packing scheme that eliminates redundant rotations by considering the entire neural network architecture. Finally, we demonstrate that HADES matches the accuracy of vanilla FL while preserving privacy and achieving optimized runtime through selective encryption.

### 论文提出的方法论

- **核心思想**：利用PCA（主成分分析）识别数据集中最能表征隐私敏感信息的特征子集（`F_HE`），对其使用MHE进行全程加密训练（权重、梯度始终加密）；剩余特征（`F_P`）使用明文网络训练；两个子网络的logits通过加权融合（score-level fusion）得到最终预测，用于指导加密网络的优化，而明文网络则使用自身的plaintext loss更新，无需解密HE参数。
- **关键技术细节**：
    - **特征选择**：将原始特征投影到PCA主成分空间，选取前`|F_HE|`个主成分作为加密特征。`|F_HE|`的选择取决于方差解释率、网络配置（如ciphertext容量`N/2`）及期望的模型效用。模块化设计，可替换为其他特征选择算法。
    - **混合模型训练（Algorithm 2）**：
        1. 初始化：所有客户端协商PCA变换，划分数据集为`X_HE`和`X_P`，初始化局部模型`W^k_HE`（加密）和`W^k_P`（明文）。
        2. 本地训练：每个客户端分别对两个子网络进行前向传播，得到logits `z_HE`（加密）和`z_P`（明文）。融合logit：`\bar{z}_HE = \alpha z_HE + (1-\alpha) HE(z_P)`（HE对`z_P`加密），明文logit为`z_P`。计算损失`L_HE = \bar{z}_HE - y`（加密域），`L_P = z_P - y`（明文域）。分别计算梯度并发送给服务器。
        3. 服务器聚合：分别对加密梯度和明文梯度进行加权平均（FedAvg）。
        4. 客户端更新：使用聚合后的梯度更新两个子网络。加密网络更新后需进行bootstrapping（刷新ciphertext level）。
    - **HE操作优化**：
        - **激活函数近似**：使用低阶多项式近似sigmoid（通过最小二乘法拟合），前反向传播均使用多项式及其导数。
        - **矩阵打包与旋转优化**：提出全局视角的packing方案，通过交替行/列主序排列、填充（padding）到2的幂次、计算multiplication area `\mu` 等方式，显著减少深层网络中冗余的旋转操作（见Algorithms 3、4）。支持单个ciphertext内编码整个mini-batch。
        - **融合机制**：使用score-level fusion（加权平均），融合参数`α=0.5`，可调。融合发生在logits层面，不暴露加密子网络的中间层表示。
- **算法流程**：Algorithm 2给出了完整的训练算法，包括初始化、本地训练、聚合和模型更新四个阶段。

### 论文的主要结论与发现

1. **隐私保护有效**：加密少量PCA主成分即可显著降低重构攻击质量（SSIM从1.0降至0.11甚至更低，LPIPS大幅上升），且随着`|F_HE|`增加，重构图像几乎丧失语义信息。
2. **模型效用保持不变**：HADES在三个数据集上的准确率与全明文FL相当，甚至略高（如SVHN从63.1%提升至70.4%），表明融合设计能有效利用明文子网络的信息。
3. **计算效率显著提升**：相比于全加密FL，HADES因加密特征数量减少而降低约18%-28%的训练时间（视配置而定），且开销随`|F_HE|`线性增长；深层网络下仍能保持约7%的性能提升。
4. **缩放性良好**：训练时间随客户端数量线性增长（因为本地计算和通信开销均与客户端数成正比）；随`|F_HE|`和网络深度/宽度增加，但增幅可控。
5. **融合机制优于纯PCA训练**：仅用PCA特征的单网络准确率明显低于HADES，尤其在高维数据集（MNIST、SVHN）上，表明丢弃剩余特征会损失关键信息。

### 优点

- **创新性强**：首次提出在FL中实现端到端加密与明文训练的双网络融合，属于选择性加密思路的开拓性工作。
- **实用性高**：在隐私和效率之间实现了现实可行的折中，避免了全加密的不可接受开销，同时提供了强隐私保护（MHE抗合谋）。
- **模块化设计**：特征选择、激活函数近似、融合方式均可替换，可扩展性强。
