# Verifiable Secure Aggregation via Dual Servers with Linear Tags in Federated Learning 

Yufei Zhou 

Abstract —Federated learning (FL) enables collaborative model training by aggregating local updates without requiring raw data sharing. However, prior studies have shown that servers can exploit gradient inversion to compromise user privacy or manipulate aggregation results, undermining the utility of the global model. To address these concerns, we propose a secure and verifiable aggregation scheme with lightweight cryptographic primitives for FL. Our method leverages pseudo-random func-tions (PRFs) and a non-colluding dual-server architecture to achieve secure aggregation with mutual server verification, while maintaining communication overhead comparable to plaintext aggregation and a constant verification tag size. Crucially, it pre-serves user privacy and achieves end-to-end secure aggregation with verification. Moreover, our scheme significantly reduces both user computation and verification overhead, making it suitable for FL with a large number of participants. For instance, with an input dimension of 20K, user computation time is reduced to 18 ms, approximately 7 × faster than OPSA, while verification time decreases to 9.5 ms, approximately 2.4 × faster than OPSA. 

Index Terms —Federated learning, verifiable secure aggrega-tion, pseudo-random function 

I. I NTRODUCTION 

Federated learning (FL) has emerged as a promising paradigm in machine learning, offering a decentralized and privacy-preserving alternative to traditional centralized train-ing frameworks [1]. In FL, users retain their raw data locally and collaboratively train a global model by sharing only model updates with a central server. The server aggregates these updates, typically using FedAvg [2], to obtain the global model. This decentralized approach addresses the challenge of data silos and enables collaborative learning across devices or institutions, particularly in sensitive domains such as health-care and finance [3]. Despite its advantages, FL introduces several critical secu-rity challenges. In its basic form, the plain FL aggregation does not fully protect user privacy or guarantee the security of the global model. This creates three major issues: 1) Privacy leakage from local model updates. Since the central server receives local model updates in plaintext, it can potentially launch gradient inversion [4] or model inversion [5] attacks to reconstruct sensitive user data. 2) Malicious or incorrect aggregation. A dishonest server could perform incomplete computations to save re-sources [6], inject backdoors into the global model [7], or intentionally return crafted aggregation results to extract private information [8].

3) Misuse and leakage of the global model. After model convergence, the server may redistribute or commercial-ize the global model without user consent, undermining user interests and data ownership [9]. Current works have proposed verifiable secure aggregation protocols, aiming to address the aforementioned issues. For example, the schemes in [6], [10], [11] primarily focus on mitigating the first two threats by employing lightweight ran-dom masking. Recent works such as [12] and [13] have made strides in addressing the third issue, combining verifiability with protection against global model leakage. However, their reliance on complex cryptographic primitives such as homo-morphic hash functions [14] or homomorphic encryption [15] results in significant computational and communication over-head, which becomes especially challenging given the large input dimension of modern deep learning model updates. While OPSA [16] reduces additional user communication for verification to O(1) , it relies on trusted execution environments (TEEs), which may not always be available and are susceptible to side-channel attacks. Thus, efficiently addressing all three issues simultaneously remains a significant challenge. Furthermore, the trustworthiness of the initial model has received limited attention in existing works. In practice, the global model is often initialized by the server, which intro-duces a potential risk of poisoning attacks from the very beginning of the training process. Such attacks can have per-sistent effects, with backdoors remaining in the final converged model [17]. Therefore, ensuring the verifiability of the entire secure aggregation process is also crucial for the security of FL systems. In this work, we propose an efficient secure aggregation scheme based on the non-colluding two-server assumption and two-party additive secret sharing. Our scheme utilizes linear tags to verify aggregation results, enabling fast proof generation and constant-size tags. Through mutual verification between the two servers, we achieve end-to-end verifiabil-ity, from model parameter initialization to final converged aggregation. Finally, by leveraging pseudo-random functions (PRFs), we optimize the communication overhead of the protocol to be comparable to that of plaintext aggregation. Our main contributions are as follows: 1) Lightweight verification with constant-size linear tags. 

We introduce a novel verification mechanism based on linear tags over integer fields, enabling fast proof gen-eration with constant-size verification overhead that is independent of both model size and user number. Our two-server architecture allows for mutual validation with-out relying on trusted hardware or authorities, improving

Verifiable secure aggregation has been widely studied. In this section, we briefly review representative works and sum-marize the key differences between our approach and existing methods. 

One Server. VerifyNet [18] is the first to address the problem of verifiable aggregation, leveraging homomorphic hash functions and ZKPs. However, it suffers from high communication overhead and slow verification time. VFL [19] uses the Chinese Remainder Theorem for verification, but it does not support user dropouts. VerSA [6] achieves verification via double masking and linear message authentication codes (MACs), but its protocol has known issues [20]. Guo et al. [21] propose a scheme based on homomorphic hashing and commitment, but the verification overhead grows linearly with the number of users. OVP-FL [22] adopts bilinear pairings for verification, resulting in high computational costs. Yang et al. [12] consider both weighted aggregation verification and global model privacy, but similarly rely on bilinear pairings. OPSA [16] achieves constant verification communication cost but depends on TEEs. 

Two Servers or More. Due to the high overhead of single-server aggregation, many recent works adopt two-server architectures or involve auxiliary nodes to perform aggregation and verification. VOSA [10] uses one server and one col-lector for verification, but its security has been broken [24].

C. Secret Sharing 

A secret sharing scheme typically consists of two polynomial-time algorithms: the sharing algorithm and the reconstruction algorithm. In the sharing algorithm, the dealer splits a secret element x ∈ ZR into two random shares 

s1, s 2 ∈ ZR, where s1 is sampled uniformly at random from ZR and s2 = x − s1 mod R, and sends them to two participants P1 and P2, respectively. In the reconstruction algorithm, the original secret x can be recovered from the shares (s1, s 2), that is, x = s1 + s2 mod R. The shares s1, s 2

are uniformly distributed and no information about x is leaked. Secret sharing typically supports additive homomorphism. Let the shares of x1 be (s1, s 2), and the shares of x2 be 

(t1, t 2). Then, the secret shares of x1 and x2 can be combined to obtain a secret share of x1+x2. Specifically, (s1+t1, s 2+t2)

A. System Model and Threat Model 

Our scheme involves three types of entities: a computation server, a verification server, and multiple users. 

Computation Server (CS) : Responsible for aggregating local models. It is provided by a third party with powerful computational and communication resources. 

Verification Server (VS) : Responsible for generating verifi-cation information and assisting in aggregation. Similar to CS, it is also provided by a third party and has powerful resources. 

Users : Hold local private datasets and can perform model training. However, due to insufficient data, they need to collaborate with other users through FL to obtain a more powerful global model. We assume that CS and VS are two non-colluding but potentially malicious entities. Each may attempt to infer users’ private data or forge aggregation results. This assumption of a non-colluding dual-server setting is commonly adopted in existing literature [9], [10], [23]. We assume that users are semi-honest , meaning they follow the protocol correctly but may attempt to infer additional information from the messages they receive. Some users may also collude with one of the servers to try to learn other users’ private data. Semi-honest users are common in practice, for example, in banks or hospitals. Existing verifiable secure aggregation schemes, such as [6], [16], also adopt the semi-honest user assumption. In addition, we assume the existence of a Public Key Infrastructure (PKI), and that communication between any two entities is conducted over secure channels such as SSL [27]. Therefore, external adversaries are unable to obtain any private information. The main security goals of our scheme are as follows: 1) Verifiability : In the absence of collusion, neither server can forge aggregation results. 2) Privacy : No server can obtain any private information of users or information about the converged global model. 3) Collusion Resistance : Even if one server colludes with a subset of users, it cannot learn any private information from the remaining users. It is worth noting that when a server colludes with certain users to forge the global model, the verification process becomes infeasible. This is because colluding users may participate in the aggregation and present forged results as legitimate model updates. Such attack scenarios are beyond the scope of this work. Therefore, in the case of server-user collusion, we only consider privacy security, in line with current related works [9], [16]. 

B. Efficient Secure Aggregation 

Our secure aggregation protocol consists of four phases: 

Setup , Share , Aggregate , and Reconstruct . In the Setup 

phase, public parameters are negotiated, and cryptographic keys are generated and distributed. During the Share phase, each user shares their models with the servers using additive secret sharing. In the Aggregate phase, the servers aggregate the received shares to obtain the shares of the global model by leveraging the homomorphic property of secret sharing. Finally, in the Reconstruct phase, users reconstruct the global model from the aggregated shares, completing one round of FL. D. Some Remarks on Our scheme 1) User Dropouts: Users may drop out during training. Our scheme tolerates arbitrary dropouts: as long as a user uploads the shares to the servers, it can participate in aggregation. Moreover, each aggregation round requires only one commu-nication round, further reducing the dropout probability. 

2) New Users During Training: Unlike many prior schemes [6], [18], our protocol does not require inter-user key negotia-tion. A new user Uo only needs to interact with the servers for initialization. Specifically, Uo first receives Kc,v , K c,g from CS and Kv,v , K v,g from VS. It then randomly samples two keys, 

Kc,o and kv,o , and sends them to CS and VS, respectively. Finally, Uo computes Kv = Kc,v ∥Kv,v to complete key generation. Afterward, Uo downloads the current global model shares and can immediately participate in the next training round. Since Uo possesses Kv , it can also verify that the current model has been correctly aggregated, ensuring model consistency across different rounds. 

3) Verification with Weighted Aggregation: When users have different aggregation weights, each user Ui incorporates its weight αi into the local model parameters in advance, computing w′ 

> i

= αiwi. The weight αi is treated as an addi-tional parameter and aggregated together with the model. After aggregation, users can recover the sum of all weights, which enables decoding of the true aggregated model parameters. 

4) Traffic Between CS and VS: In each secure aggregation round, the servers communicate in two steps. First, they exchange the IDs of the users they received shares from to determine the set of online users. Then, VS sends Wt to CS, while CS sends bt to VS. The exchanged ID lists scale linearly with the number of online users but are negligible in size. The message Wt has the same size as the model parameters, and

5) Discussion of Limitations: Our scheme requires two non-colluding servers, which makes it unsuitable for all sce-narios. However, in practice, deploying two non-colluding servers is feasible, for example, by using two different cloud providers, two distinct administrative entities, or servers lo-cated in different regions or countries. Conflicting interests and legal contracts prevent them from colluding. This assumption is consistent with most existing works on privacy-preserving computation [23], [28]–[30]. In addition, our scheme does not defend against malicious users. When a user is controlled by an adversary, the adversary can forge local model updates to manipulate the global model. Addressing this problem is beyond the scope of this work. Nevertheless, many schemes have been proposed under the same non-colluding two-server assumption to mitigate malicious users, such as [31]. Our scheme can be combined with these approaches to make the FL-trained model more robust. V. S ECURITY ANALYSIS 

A. Security without Collusion 

The security of our scheme relies on the security of the underlying secret sharing and PRF. The PRF security [26] ensures that the outputs are sufficiently random and reveal no information about the key. The secret sharing scheme [32] ensures that two servers cannot infer any user privacy from the shares. This intuition is formalized in the following theorem: 

Theorem 1. Suppose CS and VS do not collude with each other, and users do not collude with servers. If the underlying PRF and secret sharing scheme are secure, then our verifiable secure aggregation scheme does not leak any private user information. 

We prove Thm. 1 using a simulation-based approach [33]. Specifically, we construct a polynomial-time simulator such that the values obtained by the server through interaction with the simulator (i.e., the simulated view) are computationally indistinguishable from the values observed by the server in the actual protocol execution (i.e., the real view). The simulator operates independently of the users’ private data, which implies that the protocol execution does not leak any private information about the users. The detailed proof is as follows. 

Proof. Since secret shares are indistinguishable across differ-ent parties, the information obtained by CS and VS in our scheme is symmetric. Therefore, we only consider the view of CS in the following proof. A simulator for VS can be constructed analogously. The real view view c of CS in an execution of the proposed verifiable secure aggregation protocol consists of the secret shares of the user models, w1,1, w2,1, · · · , wm, 1. Note that the shares of the tags are locally generated by CS using the PRF, and thus do not reveal any additional information. Next, we employ the standard hybrid argument technique: starting from the real view, we construct a sequence of polynomial-time simulators via a series of successive modi-fications, such that each pair of adjacent hybrid distributions is computationally indistinguishable. 

SIM 0 This simulator produces a view identical to the real view view c by executing the aggregation exactly as specified in the real protocol. 

SIM 1 This simulator behaves identically to SIM 0, except that during the Share phase, each user Ui replaces wi, 2 with a uniformly sampled random value. Since the pseudo-random values generated by the PRF are computationally indistinguishable from truly uniform random values, the shares computed by Ui are indistinguishable from the original shares. Therefore, the views generated by SIM 1

and SIM 0 are computationally indistinguishable. 

SIM 2 This simulator behaves identically to SIM 1, except that during the Share phase, each user Ui replaces 7

the local model parameters wi with uniformly sampled random values. Since the secret sharing scheme is secure, the shares computed by Ui are uniformly random in 

ZRw and are indistinguishable from the original shares. Therefore, the views generated by SIM 2 and SIM 1 are computationally indistinguishable. 

SIM 3 This simulator behaves identically to SIM 2 in com-puting wi, 1, while all other operations, such as model training and the Reconstruct phase, are replaced by random behaviors. Since the values sent to CS by SIM 3

are the same as those sent by SIM 2, the views generated by SIM 3 and SIM 2 are computationally indistinguishable. Note that the computations performed by SIM 3 do not require any inputs from the users or VS, and are thus independent of the user inputs. By the transitivity of computational indistin-guishability, the view generated by SIM 3 is indistinguishable from the real view view c of CS. Therefore, CS cannot learn any private information about the users. For the verification, we have the following theorem:

B. Security of Multi-Round Aggregation 

We assume that the two servers in our scheme are non-colluding. Intuitively, because each round uses fresh pseudo-random outputs of PRF with round-specific inputs, the secret shares generated in each round are independent, so the multi-round security reduces to single-round security. More formally, we have the following theorem: 

Theorem 3. Suppose that CS and VS do not collude, and that users and servers are also non-colluding. If the underlying PRF and the secret sharing scheme are secure, then the proposed verifiable secure aggregation scheme does not leak any private information of users to the servers, even when performing multi-round aggregation. Proof. In each round r, since the inputs of PRF differ across rounds, the PRF outputs in different rounds are computation-ally independent. Each server observes only the secret share. Therefore, multi-round security reduces to single-round secu-rity: no additional information can be gained by correlating shares across rounds. From Thm. 1, single-round aggregation is secure. Since each round is an independent instance, the adversary cannot gain more information from multiple rounds. Hence, we conclude that the multi-round verifiable secure aggregation protocol preserves user privacy under the stated assumptions. In particular, servers learn nothing beyond the aggregated result in each round, and the protocol is secure across multiple rounds. 

C. Security with Collusion Between Users and One Server 

In our protocol, we consider the scenario where some users may collude with one of the servers. While in this case, the verifiability of the aggregated result for the colluding users cannot be guaranteed, the privacy of non-colluding users is still preserved as long as the two servers do not collude. Specifically, a colluding adversary (comprising some users and one server) can observe the global model information and the corresponding verification tags of the colluding users. However, the model updates of the remaining non-colluding users are secret-shared between the two servers. As long as the servers themselves remain non-colluding, the adversary cannot access additional information from these shares. Consequently, the privacy of non-colluding users is maintained, and no private information is leaked beyond what the adversary is already authorized to observe. VI. T HEORETICAL OVERHEAD ANALYSIS

1) Computational Complexity: For users, our scheme has a computational complexity of O(d), which is significantly lower than that of other schemes. Our scheme only requires a 2-out-of-2 additive secret sharing of model parameters, resulting in a computational complexity of O(d). In contrast, RVFL requires an m-out-of-m additive secret sharing of model parameters, where m denotes the number of auxiliary nodes, leading to a computational complexity of O(md ). POT incurs similar overhead as RVFL because it uses random masks from each neighbor to hide the true model parameters, resulting in a computational complexity of O(dA ), where A

In our scheme, the main overhead for the verification server is to generate shares for each user using a PRF and then perform aggregation, while the computation server directly aggregates the users’ shares. Therefore, the computational complexity of our scheme is O((1 − δ)nd ). Note that the computational complexity of the TEE in POT and KhPRF is independent of d, so their overhead is lower than that of our verification server. However, the total server overhead is comparable across schemes. 

2) Communication Complexity: Regarding the communi-cation complexity for users, our scheme is similar to KhPRF, both being O(d), which is lower than other schemes. How-ever, note that OPSA uses almost-KhPRF techniques, which require reserving extra space for aggregation errors, resulting in slightly higher actual communication overhead than our scheme. POT needs to share its key with neighbors, so its complexity is O(d + A). RVFL requires receiving key shares from auxiliary nodes, yielding a communication complexity of O(d + m).For the server-side communication overhead, the situation is similar to that of users, since the main communication occurs between servers and users. Therefore, the primary communication complexity is O((1 − δ)nd ). In our scheme, communication between the two servers involves only the sharing of aggregated models and some online user informa-tion, with complexity O(d+n). RVFL, in addition to user com-munication, also exchanges information with auxiliary nodes, resulting in complexity O((1 − δ)nd + md ). In OPSA, the two servers in each secure aggregation scheme communicate with the TEE, but this communication is relatively small. The main overhead still comes from (1 − δ)n users.

We implemented our scheme in Python, with communica-tion performed over secure sockets. Our verifiable secure aggregation achieves nearly identical accuracy to plain aggregation. VIII. C ONCLUSION 

In this work, we propose a verifiable secure aggregation scheme that enables fast aggregation and verification of high-dimensional models. By leveraging secret sharing and PRFs, we achieve communication costs comparable to those of plaintext aggregation. Additionally, we employ linear tags over integer fields to verify aggregation results, which allows for fast proof generation and constant-size tags. This approach is highly user-friendly, requiring minimal computational and communication overhead. Through mutual verification be-tween the two servers, our scheme ensures end-to-end verifia-bility for both model initialization and aggregation. Our design not only prevents servers from learning any local user data but also protects the privacy of the global model and verification tags from servers, thus enhancing privacy throughout the FL process.
