## 摘要
密集嵌入支撑着语义搜索和检索增强生成，然而泄露的向量存储会暴露大部分底层文本。现代反演和对齐攻击有一个共同弱点：受保护的存储是单一的全局几何结构，而任何单一的几何结构都可以与已知的几何结构对齐——包括秘密的全局旋转，因为正交Procrustes可以从大约子空间维度的已知明文对中恢复它。我们提出SHARD，一种保留检索的嵌入变换，消除了这一薄弱轴。中心化嵌入被旋转并分割成短公共前缀（驱动第一阶段检索）和私有残差，该残差被分片到C个单元中，每个单元在独立的密钥下旋转；残差在CKKS下重排序，其中密钥相互抵消且内积保持精确。参数C从全局线性基线（C=1）跨越到每文档微密钥（C=N），使得密钥化的残差成为文本嵌入的可取消模板——可撤销、可更新、不可链接——这是密集检索领域首个此类方案。在五个编码器上：全维重排序返回原始空间的nDCG@10，而半SVD截断会损失该指标；在弥散已知明文泄露下恢复单元密钥化残差，需要大约C倍的锚点（中位数从200到102,400，当C=256时），用于几个加密的残差查询，而短公共前缀泄露的邻居结构要少得多，微密钥极限将残差泄露降至零。该屏障对学习线性、非线性和无监督对齐器均有效，而在匹配效用的噪声防御几乎能去匿名化每个探测时，SHARD无法去匿名化任何一个。局限：单元内相似性仍然存在，针对受害者单个单元的有针对性攻击者仅需约d_priv个锚点，重叠的参考语料库仍会通过公共前缀泄露。SHARD是一种攻击感知的几何防御，而非密码学保证。

## Abstract
Dense embeddings underpin semantic search and retrieval-augmented generation, yet a leaked vector store hands much of the underlying text back. Modern inversion and alignment attacks share one weakness: the protected store is a single global geometry, and any single geometry can be aligned to a known one - a secret global rotation included, since orthogonal Procrustes recovers it from about subspace-dimension known-plaintext pairs. We introduce SHARD, a retrieval-preserving embedding transform that removes that weak axis. The centred embedding is rotated and split into a short public prefix (driving stage-1 retrieval) and a private residual sharded into C cells, each rotated under a separate secret key; the residual is reranked under CKKS, where the keys cancel and the inner product stays exact. One parameter C spans the global-linear baseline (C=1) to per-document micro-keys (C=N), making the keyed residual a cancellable template - revocable, renewable, unlinkable - for text embeddings, the first such scheme for dense retrieval. On five encoders: full-dimensional reranking returns the raw-space nDCG@10 that half-SVD truncation gives up; recovering the cell-keyed residual under a diffuse known-plaintext leak costs about C times more anchors (median 200 to 102,400 at C=256) for a few encrypted residual queries and the short public prefix leaks far less neighbour structure, with a micro-key limit driving residual leakage to zero. The barrier holds against learned-linear, non-linear and unsupervised aligners, and where a matched-utility noise defence de-anonymises almost every probe, SHARD de-anonymises none. Limits: within a cell similarities survive, a targeted attacker on one victim's cell needs only about d_priv anchors, and an overlapping reference corpus still leaks through the public prefix. SHARD is an attack-aware geometric defence, not a cryptographic guarantee.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：密集检索中嵌入向量的隐私保护面临严重威胁。现代对齐/反演攻击（如 Procrustes、ALGEN、ZSInvert）能够利用少量已知明文对恢复嵌入空间的全局几何，从而重构原始文本。现有轻量级防御（如秘密全局旋转、SVD截断）的弱点是**单一全局几何可被对齐**。
- **动机**：需要一种能抵抗对齐攻击的几何变换，使得即使泄露部分已知明文对，攻击者也无法恢复整个空间，同时保持检索质量。
- **整体含义**：提出 SHARD——一种“攻击感知”的几何防御，通过将嵌入拆分为粗粒度公共前缀和细粒度密钥化残差，消除全局可对齐轴，使对齐成本随单元数 C 线性增长，并提供可撤销、可更新、不可链接的“可取消模板”能力。

## 2. 方法论：核心思想、技术细节与算法流程
### 核心思想
- 将中心化后的嵌入经 PCA 旋转分解为**短公共前缀**（高方差方向，用于第一阶段 ANN 检索）和**私有残差**（低方差方向，携带细粒度信息）。
- 残差按公共前缀的 k-means 聚类分为 C 个单元，每个单元拥有独立的秘密正交密钥（Householder 反射的乘积），服务器仅存储密钥化的残差 shard。
- 在 CKKS 同态加密下进行重排序：客户端向活跃单元发送加密的残差查询，服务器计算密文-明文中积，**密钥自动抵消**，内积恢复精确值。

### 关键技术细节
- **分割**：嵌入 x → [u ∥ r]，u ∈ ℝ^d_pub，r ∈ ℝ^d_priv，d_pub 通常设为 d/4。
- **单元密钥**：H_c = ∏ (I − 2 w_{cj} w_{cj}^T) ∈ O(d_priv)，每个单元独立生成。
- **精确重排序**：⟨H_c r_q, z_i⟩ = ⟨H_c r_q, H_c r_i⟩ = ⟨r_q, r_i⟩，结合公共前缀得分恢复完整相似度。
- **参数 C**：从 C=1（全局线性）到 C=N（每文档微密钥）可调，微密钥下残差无匹配对，彻底消除单元内泄露。

### 算法流程
- **离线阶段**：计算中心化嵌入 → PCA 旋转 → 分割 u/r → k-means 分 C 单元 → 每个单元生成随机正交密钥 H_c → 存储公共前缀 u_i 和密钥化残差 z_i = H_{c(i)} r_i。
- **在线阶段**：查询编码 → 本地 ANN 用 u 选候选列表 → 找出候选列表跨越的活跃单元 → 对每个活跃单元发送加密的 H_c r_q → 服务器计算 ct-pt 内积返回 → 客户端解密，合并前缀得分，排序返回 top-k。

## 6. 主要结论与发现
1. **效用恢复**：SHARD 全维重排序匹配原始 nDCG@10，而半 SVD 截断损失 2–8 点。
2. **对齐阻力随 C 线性增长**：弥散已知明文攻击所需锚点数：全局密钥 200 → C=64 时 25,600 → C=256 时 102,400（e5-small）。
3. **强攻击者无效**：学习线性、非线性 MLP、无监督对齐均未超越正交 Procrustes（后者是最优的）。
4. **公共前缀泄露少**：d/4 前缀 NN-overlap@10 为 0.55，而基线暴露 half-space 为 0.76。
5. **微密钥消除残余泄露**：当 C=N，单元内相似性为零，且重新密钥后 AUC ≈ 0.5（不可链接）。
6. **对比 DP 噪声**：匹配效用下 DP 去匿名化 R@1 ≈ 1.0，SHARD 为 0.0。
7. **关键局限**：单元内相似性仍然存在；定向攻击仅需 d_priv 个锚点；重叠参考语料库可通过公共前缀泄露。

## 7. 优点
- **创新性**：首次将可取消模板思想引入文本嵌入隐私，参数 C 提供连续的安全-带宽权衡，微密钥实现零残余泄露。
- **实验严谨**：多数结论有统计置信区间（配对 bootstrap）、多编码器重复、随机单元控制，排除聚类偏差。
- **攻击者范围广**：不仅测试简单 Procrustes，还包括学习线性（ALGEN 核心）、非线性 MLP、无监督 vec2vec，且证明正交 Procrustes 是最强攻击。
- **透明性**：明确列出“非承诺”（如非密码学保证、重叠参考泄露、访问模式未保护），客观衡量局限。
- **实用性**：CKKS ct-pt 耗时约 0.2–0.5 秒/查询，活跃单元数增长亚线性于 C，部署可行。

## 8. 不足与局限
- **单元内相似性泄露**：同一单元内密钥抵消，服务器可计算精确 residual 相似度（约 0.5 的 top-10 邻居在 C=64 时仍在单元内）。
- **定向攻击成本低**：针对单个受害者单元，攻击者只需集中锚点于该单元，约 d_priv 锚点即可恢复。
- **重叠参考语料库**：若攻击者拥有与受保护集合重叠的公共语料，可通过恢复公共前缀密钥（仅需 d_pub 锚点）实现精确段落查找，绕过残余密钥。
- **访问模式未保护**：CKKS 隐藏数值，但候选 ID 列表、查询顺序暴露，需组合 PIR/ORAM。
- **文档隐私非密码学**：受保护数据库 E_rot 以明文存储在服务器，依赖于 SVD 截断和秘密旋转的启发式防御。
- **生成式反演未全面测试**：仅测试了离线 Vec2Text 校正器，且原始基线已处于重建下限；未测试针对 SHARD 残差空间训练的专用反演解码器。
- **计算资源有限**：BEIR nDCG 评估仅使用 CPU 编码，未在 100 万级 corpus 上运行全查询集；GPU 加速的端到端延迟未在真实搜索场景中测量。
