## 摘要
范围计数是地理信息系统中的核心原语。当数据分布在多个组织之间时，进行范围计数会引发严重的隐私问题。现有的隐私保护协议侧重于保护组织的数据集，但无法同时实现效率、查询隐私以及对重叠数据的准确性。典型协议以明文形式处理查询范围以实现高效的范围内点判定，因为保护查询隐私的设计依赖于昂贵的安全比较。此外，大多数工作假设组织间的数据集不重叠，这会导致在重叠场景中出现巨大误差。在本文中，我们提出了PPRC，这是首个同时满足所有隐私、效率和准确性要求的协议。PPRC有两项关键技术贡献。首先，我们设计了私有范围谓词（PRP）技术，该技术支持高效的范围内点判定，同时保护查询范围。PRP将范围判定重新表述为加密的成员测试，有效地用更快的安全乘法替代了昂贵的比较。其次，我们提出了不经意线性计数（OLC），这是一种聚合方案，能够高效且安全地聚合来自具有重叠数据的组织的部分结果。OLC仅涉及轻量级密码操作，并确保除最终范围计数外不泄露任何信息。我们从理论上分析了PPRC的准确性、效率和安全性。在真实世界和合成数据集上的实验表明，与基线协议相比，PPRC实现了高达55倍的误差减小和37倍的加速。

## Abstract
Range counting is a core primitive in geographic information systems. When data is distributed across multiple organizations, conducting range counting raises substantial privacy concerns. Existing privacy-preserving protocols focus on protecting organizations' datasets, but cannot simultaneously achieve efficiency, query privacy, and accuracy on overlapping data. Typical protocols process query range in plaintext for efficient point-in-range evaluation, since query-private designs rely on expensive secure comparisons. Moreover, most works assume non-overlapping datasets across organizations, which leads to huge errors in overlapping scenarios. In this paper, we propose PPRC, the first protocol that jointly satisfies all the privacy, efficiency, and accuracy requirements. PPRC makes two key technical contributions. First, we design the Private Range Predicate (PRP) technique that supports efficient point-in-range evaluation while protecting the query range. PRP reformulates range evaluation as encrypted membership tests, effectively replacing costly secure comparisons with faster secure multiplications. Second, we propose Oblivious Linear Counting (OLC), an aggregation scheme that efficiently and securely aggregates partial results from organizations with overlapping data. OLC involves only lightweight cryptographic operations and ensures that no information is leaked beyond the final range count. We theoretically analyze the accuracy, efficiency, and security of PPRC. Experiments on real-world and synthetic datasets show that PPRC achieves up to 55x smaller errors and 37x speedup compared to baseline protocols.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：在地理信息系统（GIS）中，范围计数（range counting）是基础操作，返回落在查询区域内的不同记录数。当数据分布在多个组织（如网约车公司、医院）时，需要进行分布式范围计数（DRC）。然而，现有方案存在三个难以调和的矛盾：
  - **效率 vs. 查询隐私**：大多数保护数据隐私的方案（Data-Private）将查询范围明文传递，效率高但泄露查询隐私；而保护查询隐私的方案（Query-Private）依赖安全比较（secure comparison），计算开销极大。
  - **重叠数据准确性**：大多数方案假设组织间数据集不重叠，但实际中记录重复普遍（如一个司机注册多个平台），直接求和导致巨大误差。精确的多方计算（MPC）方案支持去重，但代价过高。

- **整体含义**：作者形式化了“私有分布式范围计数（PDRC）”问题，要求同时实现准确去重、双边隐私、高效可扩展。该工作首次提出满足所有要求的协议PPRC，对分布式地理数据分析具有重要理论和实用价值。

## 2. 方法论：核心思想与关键技术细节

- **核心思想**：将范围评估转化为加密成员测试（避免安全比较），并使用轻量级加密草图进行去重聚合。

- **关键技术细节**：

  ### 私有范围谓词（Private Range Predicate, PRP）
  - **目标**：让每个数据持有者（DH）在不了解查询范围的前提下，计算每个记录是否在范围内的加密指示器 E(L)。
  - **流程**：
    1. **初始化**：查询用户（QU）将矩形范围 R = ([xℓ, xr], [yℓ, yr]) 的 x 轴和 y 轴值分别编码为两个布隆过滤器 BF_x 和 BF_y（长度 M，K 个哈希函数）。每个位用全同态加密（FHE）加密得到 E(BF_x)、E(BF_y)，连同哈希函数发送给每个 DH。
    2. **执行**：对每个记录 (x_n, y_n)，DH 计算：
       - E(L_x) = ∏_{k=1}^{K} E(BF_x[h_{x_k}(x_n)])
       - E(L_y) = ∏_{k=1}^{K} E(BF_y[h_{y_k}(y_n)])
       - E(L) = E(L_x) · E(L_y)
     - 若 x_n ∈ [xℓ, xr] 且 y_n ∈ [yℓ, yr]，则所有对应位为1，E(L)=E(1)；否则为 E(0)。这用 K 次同态乘法代替了昂贵的安全比较。

  ### 不经意线性计数（Oblivious Linear Counting, OLC）
  - **目标**：安全聚合各DH的加密指示器，去重后得到精确的范围计数，且不泄露中间信息。
  - **流程**：
    1. **本地编码**：每个DH使用全局哈希函数 h 将记录映射到长度 S 的线性计数（LC）草图。对每个记录，若 E(L)=E(1)，则在对应计数器上加1（同态加法）。
    2. **安全聚合**：中央聚合器（CA）对各DH的加密草图逐元素求和。然后应用两个轻量级变换：
       - 计数器索引混淆（随机置换计数器顺序）
       - 非零掩码（每个计数器乘以随机非零整数，保持零不变）
    3. **解密与估计**：QU解密后得到 LC 草图，统计零计数器个数 S'，使用标准 LC 估计器：RC = -S ln(S'/S)。
  - **优点**：用加法代替昂贵的 OR 操作（效率优化），并通过混淆和掩码防止推断攻击（隐私优化）。

- **整体协议**：PPRC 分为两个阶段：PRP（生成加密指示器）→ OLC（聚合得到计数）。通信共4轮（QU→CA→DH 和 DH→CA→QU）。

## 6. 主要结论与发现

- **精度**：PPRC在重叠数据上误差远小于其他方案。例如在Yelp上，MAE平均比RCC低55.79倍、比FM-Count低4.90倍、比LL-Count低12.69倍。随着重叠比例增大，PPRC误差稳定增长缓慢，而RCC误差飙升。
- **效率**：PPRC端到端时间比所有基线快数十倍。例如在Brightkite上，平均比RCC快9.37倍、比LC-Count快33.89倍、比TVA快30.16倍。通信成本比除RCC外的基线低数倍（但RCC精度差）。
- **可扩展性**：随数据量从10^4增至10^7，PPRC时间线性增长，在10^7记录时仅需88.7秒，而其他基线需要数百秒甚至更高。
- **参数影响**：增大草图长度S可降低误差但增加时间和通信；查询范围大小对时间影响很小（仅布隆过滤器大小影响通信）；合适选择假阳性率f_p=10^-4可平衡精度和通信。
- **通用性**：使用BFV方案时，PPRC仍保持12.4×快于RCC、17.7×快于TVA。

## 8. 不足与局限

1. **威胁模型假设**：仅考虑半诚实（honest-but-curious）敌手且假设查询用户不与CA/合谋。若存在恶意行为或合谋突破假设，安全性可能受损。

5. **未考虑动态数据更新**：协议假设数据集静态，未讨论记录增删改场景下的高效更新机制。

7. **实验覆盖可扩展**：虽包括合成数据集10^7规模，但所有实验在局域网单机模拟，未在真实多服务器分布式环境部署。网络延迟和带宽差异可能影响实际性能。
8. **基线对比限制**：TVA是精确MPC方案，其通信和时间在规模大时极高，对比中显得PPRC优势明显，但TVA提供精确结果，PPRC是概率近似（有误差）。部分应用可能更看重精确性。
