## 摘要
范围计数是地理信息系统中的核心原语。当数据分布在多个组织之间时，进行范围计数会引发严重的隐私问题。现有的隐私保护协议侧重于保护组织的数据集，但无法同时实现高效性、查询隐私性和对重叠数据的准确性。典型的协议以明文形式处理查询范围以实现高效的点在范围内评估，因为查询隐私设计依赖于昂贵的安全比较。此外，大多数工作假设组织间的数据集不重叠，这在重叠场景中会导致巨大误差。本文提出了PPRC，这是第一个同时满足所有隐私、效率和准确性要求的协议。PPRC有两项关键技术贡献。首先，我们设计了私有范围谓词（PRP）技术，该技术支持高效的点在范围内评估，同时保护查询范围。PRP将范围评估重新表述为加密的成员资格测试，有效地用更快的安全乘法替代了昂贵的比较操作。其次，我们提出了不经意线性计数（OLC），这是一种聚合方案，能够高效且安全地聚合来自具有重叠数据的组织的部分结果。OLC仅涉及轻量级密码学操作，并确保除最终范围计数外不泄露任何信息。我们从理论上分析了PPRC的准确性、效率和安全性。在真实世界和合成数据集上的实验表明，与基线协议相比，PPRC实现了高达55倍的误差降低和37倍的加速。

## Abstract
Range counting is a core primitive in geographic information systems. When data is distributed across multiple organizations, conducting range counting raises substantial privacy concerns. Existing privacy-preserving protocols focus on protecting organizations' datasets, but cannot simultaneously achieve efficiency, query privacy, and accuracy on overlapping data. Typical protocols process query range in plaintext for efficient point-in-range evaluation, since query-private designs rely on expensive secure comparisons. Moreover, most works assume non-overlapping datasets across organizations, which leads to huge errors in overlapping scenarios. In this paper, we propose PPRC, the first protocol that jointly satisfies all the privacy, efficiency, and accuracy requirements. PPRC makes two key technical contributions. First, we design the Private Range Predicate (PRP) technique that supports efficient point-in-range evaluation while protecting the query range. PRP reformulates range evaluation as encrypted membership tests, effectively replacing costly secure comparisons with faster secure multiplications. Second, we propose Oblivious Linear Counting (OLC), an aggregation scheme that efficiently and securely aggregates partial results from organizations with overlapping data. OLC involves only lightweight cryptographic operations and ensures that no information is leaked beyond the final range count. We theoretically analyze the accuracy, efficiency, and security of PPRC. Experiments on real-world and synthetic datasets show that PPRC achieves up to 55x smaller errors and 37x speedup compared to baseline protocols.

## 1. 核心问题与整体含义（研究动机和背景）
- **研究背景**：范围计数（Range Counting）是地理信息系统（GIS）中的基本操作，用于统计给定矩形区域内的不同记录数量。在现实应用中，数据往往由多个组织（如网约车公司、医院）分布持有，且数据存在重叠（如同一司机注册多家平台）。用户（如乘客、防疫人员）需要跨组织查询范围内的计数，但**数据隐私**（各组织的数据集保密）和**查询隐私**（用户的查询范围不能泄露）同样重要。
- **现有方案缺陷**：
  - **仅保护数据隐私**（Data-Private）的方案：如FedGroup、U-DPAP，以明文处理查询范围，泄露查询隐私。
  - **保护查询隐私**（Query-Private）的方案：如RCC、TVA，依赖安全比较（secure comparisons）或全同态加密（FHE），计算开销巨大，不适用于大规模数据。
  - **数据重叠问题**：大多数方案假设数据集不重叠，直接求和会导致严重高估；而支持去重的精确MPC方案（如TVA）代价极高。
- **核心挑战**：同时满足**准确性**（正确去重）、**双边隐私**（数据和查询隐私）、**高效性**（适合百万级记录）。论文首次形式化**私有分布式范围计数（PDRC）**问题，并提出PPRC协议作为解决方案。

## 2. 方法论：核心思想、关键技术细节
- **整体框架**：PPRC采用两阶段协议，由查询用户（QU）、数据持有者（DH）和中央聚合器（CA）组成。QU生成FHE密钥，将查询范围编码为加密布隆过滤器，通过CA分发给各DH；DH利用**私有范围谓词（PRP）**计算每个记录的加密范围内指示符；然后各DH构建加密的线性计数草图（LC sketch），通过**不经意线性计数（OLC）**安全聚合，最终QU解密并估计计数。

### 2.1 私有范围谓词（PRP）
- **核心思想**：将点-范围判断转化为**安全成员测试**，用加密布隆过滤器代替安全比较。
- **初始化（QU侧）**：
  - QU生成两个布隆过滤器`BF_x`和`BF_y`，分别编码x维和y维的整数范围（如`[x_l, x_r]`中的每个整数），每个布隆过滤器长度`M`，使用`K`个哈希函数。
  - 将每个比特加密为FHE密文`E(BF_x[m])`、`E(BF_y[m])`，连同哈希函数`Hx, Hy`和公共参数`pp`发送给CA，CA转发给所有DH。
- **执行（每DH侧）**：
  - 对于每条记录`o_n=(id, x_n, y_n)`，DH计算：
    - `E(L_x) = ∏_{k=1}^K E(BF_x[ hx_k(x_n) ])`（若所有位为1则结果为`E(1)`，否则为`E(0)`)
    - `E(L_y) = ∏_{k=1}^K E(BF_y[ hy_k(y_n) ])`
    - 最终指标`E(L_n) = E(L_x) · E(L_y)`
  - 优点：仅需`O(K)`次同态乘法，无需安全比较，且不泄露查询范围。

### 2.2 不经意线性计数（OLC）
- **核心思想**：在加密域实现线性计数（LC）草图的聚合，并通过轻量级变换隐藏计数器与记录的映射关系。
- **步骤**：
  1. **本地编码**（各DH）：
     - 每个DH初始化大小为`S`的全零向量`LC_i`，每个槽位加密为`E(0)`。
     - 所有DH协商一个全局哈希函数`h`（将记录映射到`{1,...,S}`）。
     - 对于每条记录`o_n`，若`E(L_n)=E(1)`则更新：`E(LC_i[h(o_n)]) += E(L_n)`（加法操作，非OR）。
  2. **安全聚合**（CA）：
     - 收集各DH的加密草图，按位置求和：`E(LC[s]) = ∑_i E(LC_i[s])`（轻量级加法）。
     - **隐私优化**：对聚合后的`E(LC)`进行（a）**计数器索引混淆**（随机排列顺序）和（b）**非零掩码**（每个计数器乘随机非零整数），这两步不改变零计数器数量。
  3. **解密与估计**（QU）：
     - 用私钥解密得`LC`，统计零计数器个数`S'`。
     - 估算计数：`RC = -S · ln(S'/S)`（标准LC估计器）。

### 2.3 理论分析
- **准确性**：给出偏差和标准误公式，取决于布隆过滤器假阳性概率`f_p`、草图大小`S`、总记录数`N`等。
- **效率**：每DH计算`O(KN)`次同态乘法；通信`O(M + I·S)`；总轮次4次。
- **安全性**：基于FHE的CPA安全性，证明在半诚实模型下敌手无法区分真实视图和模拟视图，保护数据和查询隐私。

## 6. 主要结论与发现
- PPRC是**第一个同时满足准确性、双边隐私和效率**的分布式范围计数协议。
- 相比RCC（误差大且无去重），PPRC平均**错误降低55倍**（MAE）；相比sketch变体（FM/LL/LC），错误降低4.8~12.7倍。
- 在效率上：相比TVA（精确MPC），PPRC**快37倍**；相比LC-Count等，快30~45倍。
- 通信开销：PPRC比LC-Count等低约4.8倍，比TVA低10.5倍；略高于RCC但准确性大幅提升。
- 可扩展性：在1000万记录上仅需88.7秒（使用SHE），且线性增长。
- 参数分析表明：默认选择`fp=1e-4`、`S=2k`可在准确性和通信之间获得良好平衡；查询范围大小对时间影响微弱。

## 7. 优点
- **方法创新**：
  - PRP将范围判断转化为布隆过滤器成员测试，用同态乘法替代安全比较，大幅降低计算开销。
  - OLC通过轻量级加法聚合和随机化掩码，避免了昂贵的OR操作，并防止了基于计数器值的推断攻击。
- **理论完备**：给出了准确性偏差、标准误的闭式公式；安全性基于CPA安全证明。
- **实验详尽**：与多种代表性方案（加密、MPC、sketch）对比，涵盖不同参数和数据集；验证了多种FHE实现的通用性。
- **实用性**：协议仅需4轮通信，适合局域网部署；在1000万记录上可在1.5分钟内完成，具有实际部署潜力。

## 8. 不足与局限
- **准确性损失**：布隆过滤器和LC草图均为概率数据结构，存在假阳性和估计误差，不适用于需要精确计数的场景。
- **场景限制**：仅支持矩形查询范围；虽讨论可扩展到Geohash等，但未实验验证。
- **安全性假设**：要求半诚实模型且QU不与CA/DH合谋；若QU合谋则可获得额外信息（作者已说明超出范围）。抗恶意敌手未考虑。
- **实验覆盖**：
  - 未在更大规模多方（I>20）或高延迟网络（如广域网）上测试。
  - 缺乏与联邦学习或差分隐私方案的比较（如FedGroup、U-DPAP，因为它们保护数据隐私但不保护查询隐私）。
  - 未测试多查询并发场景下的吞吐量。
- **资源局限性**：实验仅使用单机（8核CPU），未利用并行或分布式计算优化；但论文未声明未优化。
- **讨论未深入**：支持多QU、非连续空间扩展等仅简要提及，缺乏实验或形式化分析。
