## 摘要
设备端的大语言模型推理面临响应延迟、有限硬件资源和用户隐私的三难困境。全云推理提供强大的计算能力，但暴露用户提示和对话数据，而独立的设备端推理对于大多数消费类和嵌入式边缘设备来说不可行。本文提出了一种以隐私为中心、基于端点验证KV缓存的边缘云协同大语言模型推理框架。本地端点处理输入预处理、嵌入计算、自适应特征优化、KV缓存验证、推测解码和低维模型头计算，而云端则执行经过验证的解码器推理、KV缓存管理、令牌验证和高维词汇投影。端点融合部分输出，应用语言自适应掩码并采样目标令牌。所有传输数据和截断logits均经过量化和AES-GCM加密以保护隐私，核心轻量级模块、草稿参数和缓存访问策略保留在本地以避免泄漏。该框架通过优化的流式处理、批处理和量化ONNX部署，支持包括纯CPU、配备GPU和嵌入式设备在内的异构设备。评估表明，与基准分割推理相比，该框架将每令牌延迟降低高达46.1%，下行负载降低高达67.4%，同时保持与全云推理相当的性能。

## Abstract
On-device LLM inference faces a trilemma of response latency, limited hardware resources and user privacy. Full cloud inference delivers strong computing power but exposes user prompts and dialogue data, while standalone on-device inference is unfeasible for most consumer and embedded edge devices. This paper presents a privacy-centric edge-cloud collaborative LLM inference framework built on endpoint-authenticated KV cache. Local endpoints handle input preprocessing, embedding computation, adaptive feature optimization, KV cache authentication, speculative decoding and low-dimensional model head calculation, while the cloud conducts authenticated decoder inference, KV cache management, token verification and high-dimensional vocabulary projection. Endpoints fuse partial outputs, apply language-adaptive masking and sample target tokens. All transmitted data and truncated logits are quantized and AES-GCM encrypted for privacy, with core lightweight modules, draft parameters and cache access policies kept local to avoid leakage. The framework supports heterogeneous devices including CPU-only, GPU-equipped and embedded devices via optimized streaming, batching and quantized ONNX deployment. Evaluations demonstrate that the framework reduces per-token latency by up to 46.1\% and downlink payloads by up to 67.4\% over baseline split inference, retaining comparable performance to full cloud inference.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **隐私导向的分工**：端点（边缘设备）控制敏感决策和轻量计算（如嵌入、KV缓存授权、推测解码、局部投影），云端负责重型解码器推理和高维词汇投影。
- **通信最小化**：避免传输原始文本、完整对话历史和全量logits分布，通过语言自适应词汇掩码和设备自适应张量传输减少下行数据量。
- **加密与本地模块保护**：所有传输张量经量化和AES-GCM加密，端点本地保留LoRA模块参数、草稿模块参数和缓存访问策略。

### 关键技术细节
1. **端点控制KV缓存授权**：
   - 端点生成控制元组 `c = (flag_Cache, L_Cache, d_type, ℓ)`，其中 `flag_Cache` 决定是否允许使用历史KV缓存，`L_Cache` 为授权缓存长度，`ℓ` 为语言标识。
2. **设备自适应特征准备**：
   - **纯CPU端点**：特征张量按列分块（`N_b` 块）传输，实现流式处理（接收第k+1块同时解密第k块）。
   - **GPU端点**：可进行模型切片、张量并行和多会话批处理。
   - **边缘设备**：使用量化ONNX Runtime部署，支持硬件加速器。
3. **分布式词汇投影**：
   - 将隐藏向量按隐藏维度分解为 `h' = [h_end, h_cloud]`，对应LM头权重矩阵也拆为 `W_end` 和 `W_cloud`。
   - 端点计算 `z_end = h_end * W_end`，云端计算 `z_cloud = h_cloud * W_cloud + b`，然后端点相加得到logits。
   - 语言受限时，云端只返回词汇子集 `V_ℓ` 对应的logits，大幅减少通信量。
4. **端点推测解码与云端验证**：
   - 端点使用轻量草稿模块 `g_φ` 提出 `N` 个未来令牌，云端使用目标模型验证并提交接受令牌，拒绝时重新计算。
5. **传输保护与本地模块**：
   - 张量先量化到b位，再用AES-GCM加密传输。
   - 端点本地的LoRA适配模块、草稿模块参数不传输至云端，增加逆向重构难度。

### 算法流程（文字描述）
1. 端点对用户输入进行分词、计算嵌入，检测语言标识，设置KV缓存授权参数。
2. 端点根据设备类型对特征张量进行自适应处理，然后加密发送给云端。
3. 云端进行授权的解码器推理，并返回加密的隐藏状态（或分块流）。
4. 在自动回归生成循环中：
   - 端点计算局部logits（`z_end[V_ℓ]`）并生成草稿令牌。
   - 云端返回截断的云端logits（`z_cloud[V_ℓ]`）。
   - 端点融合logits，应用语言掩码，采样下一个令牌。
   - 端点将采样令牌和草稿序列发送给云端验证，云端更新KV缓存。
5. 重复直到生成终止。

## 3. 实验设计

### 数据集/场景
- 1000个提示：400中文、400英文、200混合语言。提示截断或填充至512输入令牌，生成128个响应令牌。

### 基准（Baseline）
1. **纯云推理（Cloud-only）**：端点发送原始提示和对话ID，云端返回文本令牌。
2. **纯云推理+加密传输**：同上，但通信加密。
3. **朴素分割推理（Naive split inference）**：端点计算嵌入，云端返回完整隐藏状态或全量logits，无端点缓存控制、语言自适应压缩或推测解码。
4. **端点纯轻量模型（Endpoint-only lightweight）**：仅在GPU和边缘配置下可行，使用小型本地模型。

### 评估指标
- 平均每令牌解码延迟、端到端延迟（128令牌响应）、吞吐量（令牌/秒）、每令牌下行负载、草稿接受率、任务级生成质量（Next-token agreement, Perplexity, MMLU, GSM8K, HumanEval pass@1）以及隐私暴露核算。

### 实验条件
- 网络：20ms RTT、2ms抖动、100Mbps带宽限制。
- 重复5次，取均值，标准偏差报告。
