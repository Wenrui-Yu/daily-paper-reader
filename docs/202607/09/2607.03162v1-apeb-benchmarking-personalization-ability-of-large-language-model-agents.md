## 摘要
当用户提出原始的、不明确的查询时，基于大语言模型的智能体在个性化方面面临挑战。在这种情况下，智能体必须推断潜在意图，从嘈杂的交互历史中提取偏好，并在相互竞争的选项中做出选择。现有基准测试很少检验这一能力，因为它们通常依赖于用户精炼后的查询或简化的历史记录。我们引入了个性化产品搜索（PPS），这是一个在原始查询和多样化历史记录下测试智能体个性化能力的实验平台。我们从行为日志中构建了智能体个性化基准（APeB），将不明确的意图与丰富的历史记录以及用户查看过的候选项目配对。通过多步智能体工作流评估最先进的大语言模型，我们发现模型能够很好地处理明确查询，但在需要意图和偏好发现的早期阶段查询中表现不佳。评分分析表明，这种差距主要源于对历史记录的有效利用不足。一种简单的历史感知查询精炼管道VQRA取得了持续的改进，凸显了个性化智能体中专用历史记录利用模块的必要性。

## Abstract
LLM-powered agents struggle with personalization when users issue raw, underspecified queries. In this setting, agents must infer latent intent, extract preferences from noisy interaction histories, and select among competing alternatives. Existing benchmarks rarely test this capability, as they often rely on user-refined queries or simplified histories. We introduce personalized product search (PPS), a testbed for agentic personalization under raw queries and diverse histories. We construct Agent Personalized Benchmark (APeB) from action logs, pairing underspecified intents with rich histories and user-viewed candidate items. Evaluating state-of-the-art LLMs with multi-step agent workflows, we find that models handle explicit queries well but struggle with early-stage queries requiring intent and preference discovery. Rubric analysis attributes this gap mainly to ineffective history use. A simple history-aware query-refinement pipeline, VQRA, yields consistent gains, highlighting the need for dedicated history-utilization modules in personalized agents.

## 2. 方法论

### 核心思想
- 构建一个基于产品搜索的个性化测试床——**PPS**（Personalized Product Search），从真实用户行为日志中构建基准数据集**APeB**（Agent Personalized Benchmark），包含模糊意图查询、丰富多样的用户历史记录以及用户实际查看过的候选商品。
- 设计多步智能体工作流来评估LLM的个性化能力，并引入一个简单的历史感知查询精炼管道**VQRA**（Vision-Query-Refinement Agent）作为改进基线。

### 关键技术细节
- **APeB数据集构建**：
  - 来源：真实用户行为日志（如电商平台商品点击、购买等）。
  - 结构：每个测试样本包含一个**原始查询**（underspecified intent）、一段**用户历史**（包含之前点击、购买、搜索过的商品，带时间戳和噪声）、以及一组**候选商品**（用户实际查看过的）。
  - 特点：查询不明确（如“送女朋友礼物”），历史记录丰富但存在噪声（如混合了不同兴趣的交互）。
- **多步智能体工作流**：
  - 步骤1：**意图理解**——从原始查询中推断用户潜在目标。
  - 步骤2：**偏好提取**——从用户历史中提取关键偏好（如价格区间、品牌、品类）。
  - 步骤3：**候选排序**——基于意图和偏好对候选商品进行排序或推荐。
- **VQRA管道**：
  - 通过将用户历史输入LLM，生成经过细化的查询（refined query），再结合细化查询与候选商品进行匹配和排序，实现历史感知的精炼。

## 6. 主要结论与发现

1. **模型对显式查询处理良好**：当用户查询足够明确时，LLM能够给出合理推荐。

5. **需要开发专门的历史利用模块**：当前LLM缺乏对噪声历史的有效过滤和偏好集成能力，是未来个性化智能体的关键改进方向。
