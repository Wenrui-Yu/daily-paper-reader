## 摘要
个性化改变了模型对用户说的话；我们表明它也可以改变用于证明回应合理性的推理轨迹。现代大语言模型通过存储用户属性、偏好和先前的上下文，然后将这些信息注入未来的提示中来个性化交互。我们研究这种记忆是否会重塑对不存在单一真实答案的开放式问题的推理。为了量化这种效应，我们引入了DRIFTLENS，一个无真实答案框架，将每个表达的推理步骤映射到一个价值类别，并衡量问题在无记忆轨迹与注入用户属性记忆下的轨迹之间的分歧。我们首先验证了DRIFTLENS能够区分无内容的语用噪声与实质性的推理变化。在四个大语言模型和10个用户属性类别（包括年龄、职业和残疾）中，用户属性记忆在每个模型的语用噪声底线之上诱导了中到大的推理漂移，即使最终答案仍然流畅、切题且合理。然后，我们评估了基于GRPO和DPO的后训练方法以减少漂移。两种方法都能减少漂移，但都没有统一占优；对下游能力、帮助性和指令遵循的影响取决于模型和奖励。这些结果表明，记忆诱导的推理漂移是个性化语言模型的一种可测量且仅部分缓解的失效模式。

## Abstract
Personalization changes what a model says to a user; we show that it can also change the reasoning trajectory used to justify the response. Modern LLMs personalize interactions by storing user attributes, preferences, and prior context, then injecting this information into future prompts. We study whether such memory reshapes reasoning on open-ended questions where no single ground-truth answer exists. To quantify this effect, we introduce DRIFTLENS, a ground-truth-free framework that maps each expressed reasoning step to a value category and measures divergence between a question's no-memory trajectory and its trajectory under injected user-attribute memory. We first validate that DRIFTLENS distinguishes content-free pragmatic noise from substantive reasoning changes. Across four LLMs and 10 user-attribute categories, including age, occupation, and disability, user-attribute memory induces medium-to-large reasoning drift above each model's pragmatic-noise floor, even when final answers remain fluent, on-topic, and plausible. We then evaluate GRPO- and DPO-based post-training methods for reducing drift. Both reduce drift, but neither uniformly dominates; effects on downstream capability, helpfulness, and instruction following are model-and reward-dependent. These results suggest that memory-induced reasoning drift is a measurable and only partly mitigated failure mode of personalized language models.

# 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：现代个性化大语言模型（LLM）会存储用户属性、偏好和历史上下文，并在未来交互中注入这些记忆以定制回答。然而，这种记忆不仅改变模型“说什么”，还可能改变“如何推理”——即推理轨迹。这种“符号漂移”（symbolic drift）在答案层面不可见：模型表面依然流畅、切题且合理，但内在的价值权衡和推理路径已悄然改变。
- **核心问题**：在缺乏标准答案的开放式任务（如职业建议、人际冲突、伦理权衡）中，如何客观测量、量化由无关用户属性记忆引发的推理漂移？这种漂移是否普遍存在？能否通过后训练缓解？
- **整体含义**：记忆诱导的推理漂移是个性化LLM的一种可测量但未被充分认识的失效模式，它削弱了模型在不同用户间保持推理稳定性的能力，可能在敏感决策支持场景中引入无意的偏见。

# 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

## 2.1 核心思想
- 将每个开放式问题的推理步骤映射到一个预定义的价值本体（value ontology），得到一条符号序列（轨迹）；然后计算在“无记忆”基线条件下的轨迹与在“注入用户属性记忆”条件下的轨迹之间的差异（漂移）。由于问题本身不依赖于用户属性，任何显著漂移都代表不必要的敏感度。

## 2.2 关键技术细节

### (1) 价值本体（Ontology）
- 初始来源自Huang et al. (2025)的2-3层价值观体系，经过78轮跨模型（GPT-OSS-120B, Qwen3-235B, Claude Sonnet 4.5）迭代优化，使交叉模型标注一致性从<40%提升至>83%。
- **4个根类别**：Social（人际/集体行为）、Practical（任务执行）、Protective（危害预防）、Personal（主观内在体验）。
- **11个子类别**：如Cultural respect and tradition, Social equity and justice, Efficiency, Security, Emotional depth等。
- 每个推理步骤被映射到一个（category, subcategory）符号。

### (2) 漂移度量
- **DTW（动态时间规整）**：度量两序列之间的最小对齐成本，使用离散替代成本（不同符号为1，相同为0），归一化到[0,1]。较低值表示更稳定的推理。
- **SRI（序列重复指数）**：结合编辑距离（序列顺序变化）和Jensen-Shannon散度（符号分布变化），加权平均（α=0.5），也归一化到[0,1]。
- 两个指标呈强正相关（Spearman ρ>0.93），互为构造效度验证。

### (3) 统计模型
- 使用线性混合效应模型（固定效应：扰动类别；随机效应：题目截距），以无扰动态为参考。
- 标准化效应大小 d_rel = (μ_treat - μ_noise) / σ_noise，采用10,000次聚类自助法计算95%置信区间，并使用Benjamini-Hochberg校正p值。

## 2.3 算法流程（文字描述）
1. **数据构造**：收集422个（严格集）开放、不可证伪、与用户属性无关的问答题，覆盖职业、伦理、金融、法律、医疗、日常困境等领域。
2. **生成基线轨迹**：对每个问题，在不加入任何记忆提示下，由模型生成推理-答案对（要求使用<think>标签内分步骤推理）。
3. **生成记忆下轨迹**：将各种扰动（语用噪声/用户属性/重大生活事件）作为单行前缀注入提示，模型生成响应。
4. **映射到本体**：使用单一标注模型Qwen3-235B，将每个推理步骤映射到价值符号，形成序列。
5. **计算漂移**：分别计算DTW和SRI分数，并与基线比较。
6. **后训练缓解**：
   - **GRPO**：在奖励函数中直接加入与参考轨迹的DTW差异项，配合格式奖励，通过在线强化学习优化。
   - **DPO**：构建偏好对（对记忆无扰且有帮助的回答 vs 受记忆困惑或忽视相关性的回答），使用离线偏好学习。

# 6. 论文的主要结论与发现

1. **DRIFTLENS有效**：无真实答案条件下，可区分纯语用噪声（无显著漂移）与实质性推理变化（重大生活事件产生大漂移）。
2. **记忆诱导漂移普遍存在**：在4个LLM上，10种用户属性均引起显著漂移（效应量中等至大，Cohen's d约0.35–0.98），即使最终回答仍显合理。漂移最强的属性为“转性别”和“残疾”；最弱的为“年龄”和“教育”。
3. **后训练可部分缓解但非统一占优**：GRPO和DPO均降低DTW，但效果更依赖模型和奖励设计。GRPO对Gemma2显著改善非分心率，但在Phi-4上反而降低非分心率；DPO在所有模型上稳定提升非分心率且不损害帮助性。
4. **能力与稳定性之间不存在零和**：多数后训练变体保持或轻微改善MMLU/GSM8K，但IFEval得分波动。
5. **没有“银弹”**：所有缓解方法均在稳定性、帮助性和能力间存在模型相关的权衡。

# 7. 优点：方法或实验设计上的亮点

- **无真实标签的测量框架**：针对开放式问题（占实际LLM交互约29%）而非事实性问题，解决了现有评估无法检测推理变化的缺陷。
- **严格的三层验证**：通过负控制（特异性）、正控制（敏感性）和双指标收敛性（构造效度）系统验证度量可靠性。
- **多模型多属性覆盖**：测试4种不同架构（Claude, GPT-OSS, Qwen, DeepSeek），涵盖10种隐私和公平相关属性，结果稳健。
- **本体迭代优化**：通过多模型共识、人机协作显著提升标注一致性，并提供语义等价重写验证，表明本体的稳定性而非过拟合。
- **后训练评估兼顾多方面**：不仅看漂移降低，还检查能力、帮助性、词汇多样性和人类评估，揭示真实世界部署的权衡。
- **统计严谨**：使用聚类自助法和Benjamini-Hochberg校正，控制家族错误率，避免多重比较问题。

# 8. 不足与局限

- **仅测量外部表达**：无法探测模型内部（隐藏）的认知过程，可能遗漏内隐漂移。
- **本体可能不够精细**：4类11子类固定本体可能跨领域/文化失去细粒度区分，且映射依赖单一标注模型（Qwen3-235B），存在系统性偏差。
- **实验规模限制**：仅测试小到中等模型（≤4B参数），未涉及前沿大模型（如GPT-4, Claude3 Opus）或纯白盒模型；GRPO计算成本高限制了更大K或更多轮次探索。
- **单轮交互**：不模拟实际多轮记忆累积场景（工具使用、长期记忆系统），实际部署中的漂移动态可能更复杂。
- **用户属性构造人为**：实验中使用的是固定字符串，非真实用户画像演化而来，可能过度简化了记忆上下文。
- **缓解实验未全覆盖**：未尝试对抗训练、提示敏感性训练等；DPO偏好对构建依赖较强法官模型（Qwen3-235B），可能引入裁判偏差。
- **数据源的语言和地域局限**：所有数据为英语，且主要来自西方互联网社区，跨文化泛化性未验证。
