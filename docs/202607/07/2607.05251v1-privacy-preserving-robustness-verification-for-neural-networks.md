## 摘要
神经网络验证与数据隐私本质上是矛盾的：验证需要完全访问模型参数和输入数据，但这两者正日益受到隐私法规和知识产权限制。这一矛盾使得鲁棒性验证在隐私敏感领域难以实际应用。在这项工作中，我们通过SecureCROWN填补了这一空白，这是首个用于隐私保护神经网络鲁棒性验证的框架。该框架基于安全两方计算（2PC），使模型所有者和数据所有者能够共同计算认证的鲁棒性边界——仅揭示最终结果，同时在半诚实安全模型下可证明地保护双方的私有数据。一个关键挑战是安全计算线性边界传播中的条件操作，其中依赖于数据的分支与标准安全计算协议不兼容。我们通过将条件逻辑表述为连续算术运算来消除分支。此外，我们引入了牛顿-拉夫森精化方法以提高数值稳定性。广泛的分析和实验表明，SecureCROWN严格匹配明文验证结果，同时在0.1-200秒内完成不同模型大小和通信设置（局域网/广域网）下的任务，证明了隐私保护神经网络验证的可行性。

## Abstract
Neural network verification and data privacy are inherently in tension: verification demands full access to model parameters and input data, yet both are increasingly restricted by privacy regulations and intellectual property constraints. This tension has left robustness verification impractical in privacy-sensitive domains. In this work, we address this gap with SecureCROWN, the first framework for privacy-preserving neural network robustness verification. Built upon secure two-party computation (2PC), our framework enables a model owner and a data owner to jointly compute certified robustness bounds -- revealing only the final result while provably protecting both parties' private data under the semi-honest security model. A key challenge is securely computing the conditional operations in Linear Bound Propagation, where the data-dependent branching is incompatible with standard secure computation protocols. We eliminate branching by formulating conditional logic as continuous arithmetic operations. Additionally, we introduce a Newton--Raphson refinement method to improve numerical stability. Extensive analysis and experiments show that SecureCROWN strictly matches plaintext verification results, while completing in 0.1--200s across varied model sizes and communication settings (LAN/WAN), demonstrating the feasibility of privacy-preserving neural network verification.

## 1. 核心问题与整体含义（研究动机与背景）

- **根本矛盾**：神经网络鲁棒性验证要求完全访问模型参数和输入数据，以计算认证边界（如 CROWN 方法）。然而隐私法规（如 GDPR）和知识产权保护日益严格，模型参数和用户数据不能被披露。
- **现有局限**：已有的验证方法（α,β-CROWN、CORA、Marabou 等）均假设明文集中式访问；隐私保护机器学习（如 CrypTen、Falcon）虽支持安全推理，但仅支持单次前向传播，无法完成涉及符号边界传播的鲁棒性验证。
- **本文目标**：首次将形式化鲁棒性验证与密码学隐私保证结合，提出 SecureCROWN，使模型方和数据方在半诚实敌手模型下联合计算认证下界，仅泄露最终结果（是否鲁棒），而不泄露双方的私有输入。

## 2. 方法论

### 核心思想
- 利用安全两方计算（2PC）将 CROWN 系列的线性边界传播（LBP）流程转化为双方秘密共享下的安全协议。
- **关键挑战**：LBP 中 ReLU 的松弛斜率 α 依赖于神经元的激活状态（未激活/激活/不稳定），存在数据依赖的分支，直接使用安全比较代价高昂；且深度递归中固定点精度误差会累积。
- **解决方案**：
  1. **分支消除**：将分段逻辑用统一的算术表达式替代：  
     \[
     \alpha = \frac{\text{ReLU}(\bar z)}{\text{ReLU}(\bar z) + \text{ReLU}(-z) + \epsilon_s}
     \]
     该表达式自动覆盖三种情况，无需分支。利用函数秘密共享（FSS）高效实现 ReLU。
  2. **安全截距累积**：类似地，将符号条件转化为 ReLU 运算的矩阵乘法：  
     \[
     \delta = -\text{ReLU}(-A)\cdot \text{ReLU}(-z)、\quad \bar\delta = \text{ReLU}(A)\cdot \text{ReLU}(-z)
     \]
  3. **牛顿-拉夫森精化**：用于改进安全除法（SecRecip）的数值精度，每次迭代近似将精确位数翻倍。

### 关键技术细节
- **协议流程**：
  - 输入秘密共享：模型方共享参数，数据方共享查询（x₀, ε, 差异向量 d）。
  - 逐层计算预激活边界（通过 Algorithm 1 ΠBackward）和松弛斜率（Algorithm 3 Πα）。
  - 最终调用 Backward 得到认证下界 ⟨f_{y,j}⟩，仅输出给数据方。
- **安全原语**：基于加法秘密共享（ASS）、Beaver 三元组、函数秘密共享（FSS）实现安全乘法、截断、ReLU、绝对值、倒数等。
- **复杂度**：通信轮次 O(L²)，通信量 O(L²d²k)，计算量 O(L²d³)，其中 L 为层数，d 为最大宽度，k = 64 位。

## 6. 主要结论与发现

1. **保真度**：SecureCROWN 在所有测试中保持 100% 的验证一致性，MRE 从 ~10⁻⁷（浅层）增至 ~10⁻³（10 层），但始终未改变验证决策。
2. **在线效率**：端到端延迟 0.1 s ~ 200.5 s，计算阶段占主导，通信量 0.77 MB ~ 604 MB。RTT 比带宽影响更大，验证了 O(L²) 轮次的性质。
3. **预处理成本**：一次性，数据无关，可在多查询间摊销。
4. **稳定性常数**：εs 在 10⁻⁴~10⁻¹ 范围内不影响验证一致性，但 MRE 随 εs 线性增加。
5. **可行性证明**：首次实现了可接受开销下的隐私保护鲁棒性验证，为后续工作奠定了基础。

## 7. 方法亮点

- **原创性**：第一个将完整鲁棒性验证（而非仅推理）与 2PC 结合的工作。
- **分支消除**：巧妙利用 ReLU 的数学性质，将分段条件转化为统一算术，避免安全比较，降低通信和轮次。
- **数值精度保障**：牛顿-拉夫森精化 + 小稳定常数，在深度网络下仍保持决策一致性。
- **模块化设计**：各安全子协议（Πα、Πδ、ΠBackward）可独立替换，易于扩展到新激活函数。
- **详细的理论分析**：给出了安全性证明、复杂度推导、固定点误差累积的界（包含稳定/不稳定 regime）。

## 8. 不足与局限

- **架构局限**：当前仅支持全连接 ReLU 网络。扩展到 CNN 需处理卷积稀疏性（通信激增），扩展至注意力层需先有成熟的 CROWN 式松弛（仍为开放问题）。
