## 摘要
隐私是用户在与AI聊天机器人交互时面临的一个重要挑战，因为用户可能明确或隐含地分享敏感信息，而AI聊天机器人可以利用这些信息进行用户画像。本文旨在通过用户端机制来保护用户隐私，该机制转换用户提示中的敏感信息，同时保留足够的信息以从聊天机器人获取有用的响应。这种方法面临保护隐私（即避免画像）和保持效用（即获得个性化和任务特定响应）之间的固有权衡。为此，我们考虑、评估并比较了四种不同的混淆操作：删除、抽象、替换以及我们引入的一种新颖的加噪/去噪方案。其他新颖的见解包括：利用数据类型分类法来识别和混淆敏感信息，并在做出混淆决策时明确考虑聊天响应的效用。首先，我们系统地独立优化并评估每种混淆操作在隐私-效用权衡方面的表现。其次，我们提出了PROMPTPET，这是一个基于LLM的代理，它使用一种受强化学习启发的规则优化器（首次应用于此场景）为提示的每个敏感部分选择最佳混淆操作。使用真实世界聊天数据集，我们展示了PROMPTPET能够达到任何单一混淆操作所能实现的最佳隐私-效用权衡，并且显著优于先前的最先进方法。

## Abstract
Privacy is an important challenge when users interact with AI chatbots, since users may share sensitive information, explicitly or implicitly, and AI chatbots can use this information for user profiling. In this paper, we aim to protect user privacy via a user-side mechanism that transforms sensitive information in a user prompt, while preserving enough information to elicit a useful response from the chatbot. This approach faces an inherent tradeoff between protecting privacy (i.e., avoiding profiling) and preserving utility (i.e., getting personalized and task-specific responses). To that end, we consider, evaluate, and compare four different obfuscation actions, namely redaction, abstraction, replacement, and a novel noising/denoising scheme that we introduce. Additional novel insights include: utilizing a data type taxonomy to both identify and obfuscate sensitive information and explicitly taking into account the utility of chat responses in making the obfuscation decision. First, we systematically optimize and evaluate each obfuscation action independently in terms of the privacy-utility tradeoff it achieves. Second, we propose PROMPTPET, an LLM-based agent that selects the best obfuscation action for each sensitive part of the prompt, using a reinforcement-learning inspired rule optimizer, applied for the first time in this context. Using a real-world chat dataset, we show that PROMPTPET matches the best privacy-utility tradeoff attainable by any single obfuscation action and significantly outperforms prior state-of-the-art approaches.

## 一、核心问题与整体含义（研究动机与背景）

- **研究动机**：用户与AI聊天机器人（如ChatGPT、Gemini、Claude）交互时，会通过自然语言透露大量敏感个人信息（如健康、财务、职业等），这些信息可能被服务商用于用户画像、广告定向或其他用户不知情的用途。现有隐私保护措施（如PII脱敏）无法应对非PII敏感信息（如兴趣、健康状况）的泄露，且无法阻止隐式推断。用户侧在数据到达服务商之前进行混淆是最强隐私增强技术（PET），但面临隐私保护与响应实用性之间的固有权衡。

- **整体含义**：本文旨在提出一种用户侧的提示词混淆机制，在保护隐私（防止画像）的同时，尽可能保留响应效用（获得个性化、任务相关的回答）。核心挑战在于：1）敏感信息对响应的贡献不同；2）敏感信息可能隐式表达；3）需要协调多种混淆操作。本文通过引入数据类型分类法、重要性评分、四种混淆操作（包括新颖的加噪/去噪方案）以及基于LLM的强化学习规则优化器来解决这些挑战。

## 二、方法论：核心思想、关键技术细节、算法流程

### 2.1 整体框架（PromptPET）
- **位置**：运行在用户设备上，位于用户与在线AI代理之间，对用户提示进行混淆后发送给AI，并对AI响应进行过滤。
- **流程**：敏感信息检测 → 重要性评分 → 混淆动作选择 → 执行混淆 → 发送请求 → 响应过滤。

### 2.2 敏感信息检测
- **敏感分类法**：采用IAB受众分类法（Demographics & Interests分支，含36个类别、656个数据类型），定义类别（Category）和数据类型（Data Type）。
- **敏感单元检测**：使用指令微调的LLM（Gemma3:12B）识别提示中的敏感单元，并映射到分类法中的类别和数据类型。
- **重要性评分**：将每个敏感单元的重要性量化为[0,1]之间的分数，反映其对生成有用响应的贡献程度。采用零样本LLM（基于实体显著性预测任务）实现。

### 2.3 四种混淆操作
- **Redact（删除）**：用占位符替代敏感单元，彻底移除信息，隐私增益最大但效用损失也最大。
- **Abstract（抽象）**：泛化为分类法中更高级的节点（如将“抑郁症”抽象为[健康]），中等隐私-效用权衡。
- **Replace（替换）**：用同一类别下不同数据类型的合理替代值替换（如将“山景城”替换为“帕洛阿尔托”），丢弃真实值但保留结构。
- **Noise（噪声/去噪）**：**本文新颖方案**。保留原始敏感单元，同时添加来自不同数据类型的诱饵信息（如将“治疗师”与“牙医”并列提问），诱导AI分别回答，然后过滤掉诱饵部分，从而在不丢失效用的情况下增加画像难度。去噪采用硬编码解析器，无需额外LLM调用，正确恢复率达97.6%。

### 2.4 规则优化（Rule Optimization）
- **方法**：受强化学习启发的优化框架（OPRO），两个LLM智能体协作：**混淆动作决策器**根据决策规则集为每个敏感单元分配动作；**规则优化器**基于验证集上的隐私-效用指标迭代改进规则集。
- **优化目标**：`α·隐私 + (1-α)·效用 - λ·|R|`，其中λ=1e-5惩罚规则长度，鼓励简洁可解释的规则。
- **流程**：种子规则集 → 验证集评估 → Top-5规则集维护 → 每轮采样小批量 → 计算每条规则的质量比（预测动作与最优动作的得分比） → 规则优化器生成新规则集。
- **种子规则集**：实验了三种种子：空集、阈值规则（重要性>0.8用Noise，否则Abstract）、Oracle规则（基于每查询最优动作分布）。

## 六、主要结论与发现

1. **重要性评分有效**：高重要性单元删除导致效用下降约是低重要性的4-5倍，验证了评分区分能力。
2. **噪声操作主导**：在四种操作中，Noise在大多数情况下最优（τ=1.0时占58.3%），且使73.7%的查询达到高隐私高效用区域，远优于删除类操作（39-42%）。
3. **无单一操作绝对最优**：即使Noise占优，其他操作在特定场景（如低重要性、搜索类查询）仍是最优，表明需要选择性混淆。
4. **规则优化显著提升**：PROMPTPET通过学习规则集，相比任何单操作策略，在全隐私-效用曲线上达到上包络线，匹配最佳单操作。
5. **超越现有基线**：相比Ngong等人和Zhou等人的方法，PROMPTPET每单位效用成本获得3.3倍隐私增益，总体隐私水平达0.78（基线最高0.55），提升1.4倍。
6. **失败模式**：主要失败为隐私遗漏（17.5%），源于隐式上下文信号未被检测；其次为效用崩溃（7.6%），源于对高重要性单元使用删除操作。

## 七、优点

- **新颖的噪声/去噪机制**：首次针对AI助手交互设计加噪策略，通过诱饵+过滤保留原始效用，避免删除类操作导致的效用损失。
- **细粒度的敏感信息建模**：利用IAB分类法将自然语言敏感单元映射到标准类别/数据类型，支持结构化混淆决策。
- **重要性评分指导混淆**：不是所有敏感单元同等重要，通过评估对响应的贡献实现差异化保护。
- **基于强化学习的规则优化**：将混淆动作选择转化为可优化问题，自动学习决策规则，无需手工设计，可解释且适应性强。
- **全面实验验证**：从单操作分析到整体比较，提供了丰富的隐私-效用分布可视化，并进行了详细的失败分析。
- **可部署性**：用户侧模型仅需12B量化模型+消费级GPU，延迟可接受（17秒中位数），不依赖服务商合作。

## 八、不足与局限

- **实验覆盖有限**：仅使用WildChat-1M单数据集，可能无法代表所有用户交互场景；且仅处理单轮查询，未扩展至多轮对话（作者指出是未来方向）。
- **对手模型假设**：假设服务商“诚实但好奇”，不主动反混淆；未考虑跨提示纵向关联攻击（如检测用户一致性）或主动诱导提问的对抗行为。
- **分类法依赖**：保护范围限于IAB分类法，若服务商使用不同分类或自由文本画像，则保护效果未知。
- **度量偏差**：隐私和效用完全依赖LLM评估（GPT-4.1进行画像和NLI），评估者本身有随机性和偏见；缺乏人工验证。
- **延迟问题**：添加11.6秒本地延迟，虽然可接受，但对实时交互场景仍有影响；检测步骤是主要瓶颈，文中未足够强调优化建议。
- **未考虑资源消耗细节**：未报告API调用次数、总GPU小时等，难以评估实际部署成本。
- **公平性局限**：基线方法（Ngong、Zhou）本不针对如此宽泛的敏感范围，且在实验中被约束了检测阶段（统一使用本文检测），可能对其不利。
