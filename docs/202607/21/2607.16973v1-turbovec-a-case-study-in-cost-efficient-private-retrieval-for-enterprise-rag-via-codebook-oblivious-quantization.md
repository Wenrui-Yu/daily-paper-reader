## 摘要
检索增强生成（RAG）系统日益为企业级大语言模型应用提供动力，然而向量检索层引入了两个尚未充分探索的挑战：（1）已训练的码本量化器可能在索引构建过程中暴露语料库统计信息，在多租户部署中造成泄露通道；（2）针对租户隔离的事后过滤会降低选择性查询的召回率。我们研究TurboVec——一个基于TurboQuant（一种无需依赖语料库训练的免训练码本标量量化器）构建的开源向量索引。在DBpedia OpenAI嵌入基准测试（d=1536，10万至99.9万个向量）上，相同内存预算下，TurboQuant 4位在所有规模上相比已训练的FAISS乘积量化在Recall@5上高出8.5至8.9个百分点。与HNSW（R@5=0.991）和IVF-PQ（R@5=0.840）相比，TurboQuant占据了一个独特的设计点：无需训练即可实现高于IVF-PQ的召回率，同时内存消耗仅为HNSW的1/4至1/8。部署在Snowpark Container Services上，TurboVec在10万个向量上实现了11毫秒的中位查询延迟，而数据仓库暴力扫描为707毫秒。内核级白名单过滤在10至1000个租户工作负载下保持了0.86至0.93的Recall@10，而事后过滤基线仅为0.09至0.19。免训练码本设计将成员推理准确率降至接近随机（50.0%），而PQ码本为57.3%。局限性包括单一数据集评估、未压缩HNSW比较以及仅基于合成数据的隐私评估。

## Abstract
Retrieval-Augmented Generation (RAG) systems increasingly power enterprise LLM applications, yet the vector retrieval layer introduces two underexplored challenges: (1) trained codebook quantizers may expose corpus statistics during index construction, creating a leakage channel in multi-tenant deployments, and (2) post-hoc filtering for tenant isolation degrades recall on selective queries. We study TurboVec, an open-source vector index built on TurboQuant - a codebook-oblivious scalar quantizer requiring no corpus-dependent training. On the DBpedia OpenAI embeddings benchmark (d=1536, 100K-999K vectors), TurboQuant 4-bit outperforms trained FAISS Product Quantization at the same memory budget by 8.5-8.9 percentage points in Recall@5 across all scales. Compared to HNSW (R@5=0.991) and IVF-PQ (R@5=0.840), TurboQuant occupies a distinct design point: higher recall than IVF-PQ without training, at 4-8x less memory than HNSW. Deployed on Snowpark Container Services, TurboVec achieves 11ms median query latency at 100K vectors versus 707ms for warehouse brute-force scan. Kernel-level allowlist filtering maintains 0.86-0.93 Recall@10 across 10-1000 tenant workloads versus 0.09-0.19 for post-filter baselines. Codebook-oblivious design reduces membership inference accuracy to near-random (50.0%) versus 57.3% for PQ codebooks. Limitations include single dataset evaluation, uncompressed HNSW comparison, and privacy evaluation on synthetic data only.

## 1. 核心问题与整体含义（研究动机和背景）
企业级检索增强生成（RAG）系统广泛使用向量检索层，但在多租户共享基础设施部署中面临两大挑战：
- **码本泄露风险**：传统量化方法（如乘积量化PQ）需要基于语料库训练码本，码本会编码语料库统计信息，可能被恶意租户利用进行成员推断或嵌入近似重建。
- **租户隔离降低召回**：搜索后过滤（post-filter）方式需要“多取再过滤”，在选择性查询上会浪费计算并降低召回率。

本文旨在通过免训练码本量化（codebook-oblivious quantization）与内核级过滤设计，同时提升检索效率、降低内存占用并增强隐私保护，为多租户企业RAG提供一个实用、低成本的检索方案。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- **TurboQuant**：一种码本无关标量量化器，其量化边界不依赖于语料库训练，而是基于已知的高维L2归一化嵌入经随机旋转后的边际分布（Beta/高斯）解析推导。
- **TurboVec**：基于TurboQuant构建的开源向量索引，支持扁平扫描（flat scan）和SIMD加速，并集成**内核级多租户允许列表过滤**。

### 关键技术细节
1. **TurboQuant流水线**：
   - 归一化：剥离L2范数，存储为标量。
   - 随机旋转：乘以固定随机正交矩阵R，使各坐标近似独立同分布（渐近趋于N(0,1/d)）。
   - Lloyd-Max标量量化：从已知边际分布预计算最优量化边界（2位/4位）。
   - 位打包：每个坐标编码为小整数，1536维向量从6144字节压缩至768字节（4位）。
   - 长度重归一化评分：使用每向量标量校正因量化导致的点积低估。
2. **TQ+校准（可选）**：拟合每坐标的平移和缩放参数以对齐经验分位数，引入有限的数据依赖（仅一阶和二阶矩），远弱于PQ的k-means质心。
3. **索引类型**：TurboQuantIndex（扁平扫描，位置寻址）和IdMapIndex（外部ID映射，支持O(1)删除和ID级访问控制），均支持在线插入、持久化、SIMD内核（NEON/AVX-512BW）。
4. **内核级多租户过滤**：在搜索内核中传入允许列表（allowlist），以32向量块为粒度，提前跳过不允许的块，避免“多取再过滤”，减少IO并保持召回。
